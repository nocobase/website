上週，我們發表了一篇題為「[GitHub Stars 最多的 12 大開源無程式碼工具](https://www.nocobase.com/en/blog/the-top-12-open-source-no-code-tools-with-the-most-github-stars)」的文章，獲得了許多好評。不少朋友留言詢問，為什麼像 n8n 和 Appsmith 這些知名產品沒有上榜。原因是我們這份榜單是基於 GitHub 上的「no-code」標籤，而這些產品僅被標記為「low-code」。

在今天的文章中，我們將回顧低程式碼專案。需要注意的是，無程式碼與低程式碼產品之間存在一些重疊（[因為這些概念確實有相似之處](https://www.nocobase.com/en/blog/what-is-no-code)）。在上一篇文章的基礎上，我們將增加三個額外的維度來全面評估這些專案：

1. 開源授權
2. 活躍貢獻者人數
3. 是否接受外部 PR

在探索 GitHub 上最受歡迎的開源低程式碼專案時，我們發現了幾個引人注目的工具。

**[NocoBase](https://www.nocobase.com/en/plugins)** 憑藉其高度可擴展的外掛程式架構，使企業能夠快速建置和擴展應用程式。

**[Supervision](https://supervision.roboflow.com/latest/how_to/detect_and_annotate/)** 專注於簡化電腦視覺模型的訓練、部署和監控，提供高效的資料標註和模型優化工具。

**[Refine](https://refine.dev/)** 提供對 React 程式碼的完全控制，非常適合建置和維護企業級內部工具，強調安全性和模組化。

**[n8n](https://n8n.io/)** 是一個開源的工作流程自動化工具，透過視覺化程式設計簡化複雜的系統整合和資料處理。

這些工具為開發者提供了多種解決方案，幫助他們快速建立強大的應用程式。現在，讓我們仔細看看這些低程式碼專案吧！

**介紹**

APIJSON 是騰訊開源的一個自動化 ORM 函式庫，提供零程式碼的後端介面和文件，讓前端開發者可以自訂回傳的 JSON 資料和結構。

APIJSON 不是傳統的低程式碼平台；它透過自動建立 API 和文件，無需編寫後端程式碼，從而簡化了後端開發。這符合低程式碼的原則，減少了所需的編碼量。然而，它更專注於透過 JSON 簡化資料庫互動，目標使用者是前端開發者。

**功能特色**

* **零程式碼實現：** 自動化 CRUD 操作，無需手動編寫後端程式碼或文件。
* **高安全性：** 自動驗證權限和管理版本，防止 SQL 注入。
* **強大的靈活性：** 支援多種查詢，包括跨資料庫連接和巢狀子查詢。

**使用案例**

APIJSON 適用於中小型前後端分離專案，例如新創公司、內部專案以及需要快速開發和迭代的應用程式，特別是那些涉及複雜資料庫操作的專案。
