## AI 員工

過去一年，AI 技術飛速發展，大型模型在許多領域展現出卓越甚至驚人的能力。

作為 AI 的重度使用者，我們將 AI 以 **AI 員工** 的形式直接引入 NocoBase 2.0——它們是您系統內工作的智慧同事。

NocoBase 中的 AI 員工 **不是聊天機器人**，也不是孤立的代理。

它們**無縫整合**到您的業務系統中，能夠理解上下文並直接執行任務。

* 它們會出現在您需要的地方——在表格和表單旁邊，隨時準備提供協助。
* 它們可以總結郵件、分析數據、組織非結構化內容並自動填寫表單、設計數據模型，甚至編寫 JavaScript 程式碼。
* 它們會自動檢索上下文數據和頁面結構，以智慧執行任務。
* 每個 AI 員工都擁有獨特的技能、工具和知識庫。
* 您可以創建無限數量的 AI 員工，組成您自己的 AI 團隊。

## 全新的 2.0 頁面系統

雖然 NocoBase 一直以來都是基於插件，但 1.x 版本在處理前端動態互動、靈活佈局或自定義邏輯時存在限制。

要在沒有顯著成本的情況下實現這些需求是很困難的。

NocoBase 2.0 引入了**完全重構的頁面系統**，旨在高效支援個性化和複雜的前端需求。

### RUNJS

**RunJS** 提供了一種輕量級的方式來擴展和自定義 UI 邏輯或樣式——無需創建插件或修改原始碼。

在頁面設計器中，您可以直接編寫 JavaScript 來：

* 自定義渲染內容（欄位、區塊、列、項目等）
* 定義自定義互動（按鈕動作、事件觸發）
* 結合上下文數據實現動態行為

RunJS 支援於：

* JS 區塊
* JS 動作
* JS 欄位
* JS 項目
* 聯動規則
* 事件流程

[了解更多關於 RUNJS](https://v2.docs.nocobase.com/interface-builder/runjs)

![20251102213506-2025-11-02-21-35-08](https://static-docs.nocobase.com/20251102213506-2025-11-02-21-35-08.png)

### 事件流程

在 NocoBase 1.x 中，聯動規則系統允許有限的邏輯編排，且局限於單個區塊。

NocoBase 2.0 引入了**事件流程**，這是前端邏輯配置的一項重大增強。

一個事件流程包含：

* **觸發事件**，例如渲染前、點擊時或值變更時
* **觸發條件**
* **動作步驟**，例如顯示通知、刷新目標區塊或執行 JavaScript

[了解更多關於事件流程](https://v2.docs.nocobase.com/interface-builder/event-flow)

### 網格佈局

在 1.x 版本中，區塊和欄位使用全寬靈活拖拽佈局，限制了精確度和美觀性。

NocoBase 2.0 引入了**新的網格佈局系統**，允許每個元素對齊網格單位，實現更精細的控制。

![20251031093341-2025-11-02-21-35-08](https://static-docs.nocobase.com/20251029201501.gif)

### 可複用區塊與彈窗

在 1.x 中，區塊的複製/引用功能有限且容易出錯，並且無法引用整個彈窗。

NocoBase 2.0 完全重新定義了這一點——實現了區塊和彈窗的**靈活引用和複製**，大幅減少了配置工作量。

## FlowEngine

2.0 中的新頁面系統由 **FlowEngine** 驅動——這是一個結合了**模型**和**流程**的下一代低代碼前端引擎。

它簡化了前端邏輯，增強了可複用性，並提高了可維護性。

通過利用流程的可配置特性，它將無代碼編排帶到了前端組件和業務邏輯中。

在 1.x 中開發插件前端通常涉及複雜的挑戰。有了 FlowEngine，這些問題現在大大簡化了。

[了解更多關於 FlowEngine](https://v2.docs.nocobase.com/flow-engine/what-is-flow-engine)

## 多應用與多空間

1.x 版本支援基本的多應用功能，但所有應用共享一個進程，無法單獨監控或管理。

NocoBase 2.0 引入了**全新的多應用與多空間架構**。

* **多應用插件** *(開發中)*
  支援動態創建和管理多個獨立應用，無需單獨部署。
  每個應用作為獨立實例運行，擁有自己的數據庫、插件和配置——可以在隔離或共享進程中運行。
  父應用可以監控和管理子應用的資源使用情況和狀態，非常適合多租戶場景。
* **多空間插件**
  允許在單個實例內進行邏輯隔離——非常適合多分支或多工廠等場景。
  業務邏輯、工作流程和配置保持一致，而數據則完全隔離。

[了解更多關於多空間](https://v2.docs.nocobase.com/multi-app/multi-space)

## 新的數據可視化

1.x 的圖表插件基於 Ant Design Charts 和 ECharts，適用於簡單場景，但在複雜、互動或聯動可視化方面存在限制。

NocoBase 2.0 引入了**全新的數據可視化插件**，完全基於 ECharts 構建，支援：

* 可視化配置模式
* SQL 和 JSON 模式
* 事件驅動的互動
* 多圖表聯動和下鑽

如果 SQL 或 JSON 看起來太複雜——您的 AI 員工 **Nathan** 可以幫忙。

[了解更多關於數據可視化](https://v2.docs.nocobase.com/data-visualization)

## 同步數據庫結構與更改欄位類型

在 1.x 版本中，只能通過 NocoBase 的數據源管理器創建表和欄位，並且之後無法更改欄位類型。

NocoBase 2.0 現在允許：

* 從主數據庫同步現有的表和欄位
* 創建後更改欄位類型

這意味著您可以直接使用現有的數據庫模式作為主要數據源。

## 從 1.x 升級到 2.0

從 1.x 升級到 2.0 很簡單——只需按照[官方指南](https://v2.docs.nocobase.com/get-started/upgrading/docker)操作。

但是，請注意 **NocoBase 2.0 目前處於 alpha 階段**，因此可能會有些不穩定。

得益於 NocoBase **UI 與數據分離的架構**，您現有的 1.x 應用將完全保持不變。

在創建新頁面時，您現在可以選擇 **v1** 或 **v2** 類型——選擇 v2 即可使用新的 NocoBase 2.0 功能。
