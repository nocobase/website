## **#1: Odoo ⭐️ 42.6k**

![Odoo.PNG](https://static-docs.nocobase.com/1280X1280%20(36)-1ff7hs.PNG)

* 開源發布時間：2014 年（在 GitHub 上）
* GitHub 星星數：42.6k
* GitHub：[https://github.com/odoo/odoo](https://github.com/odoo/odoo)
* 網站：[https://www.odoo.com](https://www.odoo.com)

**概述**

Odoo 最初於 2005 年以 OpenERP 之名推出，現已成為全球最成熟且廣受認可的開源 ERP/CRM 解決方案之一。Odoo 採用模組化但完全整合的架構，幾乎能處理所有關鍵業務功能，包括銷售、採購、庫存、財務、製造、專案管理、人力資源、行銷和電子商務。

Odoo 的後端主要基於 Python 和 PostgreSQL 建構，而前端則使用內部開發的自訂框架。它擁有龐大的插件和社群應用程式生態系統、全面的 REST API 支援以及強大的自訂能力——這使其成為需要高度客製化業務流程的企業和開發團隊的理想選擇。

**主要優勢**

* **全面的模組化設計：** 從 CRM 和 ERP 到網站管理、電子商務和行銷，Odoo 模組之間無縫整合。
* **活躍的社群與生態系統：** 超過 30,000 個社群驅動的應用程式和插件、詳盡的文件說明以及活躍的論壇。
* **廣泛的自訂能力：** 進階功能包括自訂表單、工作流程和報表工具，並搭配完善的 API 以滿足客製化的業務需求。
* **現代化的使用者介面：** 在開源 ERP/CRM 工具中，擁有領先業界的前端和行動友善的使用者體驗。
* **靈活的部署選項：** 可部署於自託管、私有雲或公有雲，官方也提供可選的託管服務。

**適合對象**

* **中大型企業：** 適合流程複雜、需要跨部門協作和整合管理的組織。
* **自訂開發團隊：** 非常適合需要大量客製化和插件開發的技術團隊。
* **漸進式系統擴展：** 企業可以從 CRM 開始，逐步採用 Odoo 模組，最終擴展到全面的 ERP 套件。

**開源狀態** Odoo 提供兩個不同的版本：

* **社群版（完全開源，LGPL v3）：** 所有原始碼均可自由取得，適合能夠自行管理整合和客製化的企業。
* **企業版（部分專有）：** 包含進階報表、線上編輯、進階行動功能以及僅能透過付費授權取得的特定插件，不提供開源存取。

對於主要依賴核心功能且能夠自行處理客製化的組織來說，社群版是可靠的選擇。需要全面進階功能的公司則必須評估採用混合方法——將免費的社群功能與付費的企業功能結合使用。

## **#2: Twenty ⭐️ 27.7k**

![twenty.png](https://static-docs.nocobase.com/c8cbf515-e336-4016-b56d-1ff3ed2fe081-15dra6.png)

* 在 GitHub 上發布：2023 年
* 星星數：27.7k
* GitHub：[https://github.com/twentyhq/twenty](https://github.com/twentyhq/twenty)
* 網站：[https://twenty.com](https://twenty.com)

**概述**

Twenty 是一個時尚、現代的開源 CRM，專為簡潔、協作和客製化而設計。它採用 Node.js、React 和 PostgreSQL 技術棧，涵蓋了潛在客戶、客戶管理、聯絡人和合約追蹤等基本 CRM 功能。

其自動化工作流程引擎、高度適應性的資料模型和全面的開放 API 尤其出色，非常適合追求快速客製化開發和整合的技術團隊。

**主要優勢**

* **現代化介面：** 簡潔、極簡的 UI，提供可與領先 SaaS CRM 平台媲美的使用者體驗——直觀且易於學習。
* **自動化與客製化：** 廣泛支援自訂資料模型、動態工作流程和自動化腳本，以適應複雜的業務場景。
* **API 驅動的架構：** 豐富的開放 API 便於進行客製化和順暢的第三方整合。
* **快速部署：** 客戶管理、聯絡人清單、銷售追蹤、合約和專案等核心模組開箱即用，確保快速上線。

**適合對象**

* **技術型團隊：** 適合擁有內部開發資源、尋求廣泛客製化和自動化能力的新創公司和成長型企業。
* **輕量級 CRM 需求：** 優先考慮銷售和客戶管理數位化效率，而不需要傳統 ERP 複雜性的公司。
* **自託管與注重隱私的企業：** 因嚴格的資料隱私、安全或合規原因而需要自託管解決方案的組織。

**授權概覽**

Twenty 採用 Business Source License 1.1 (BSL-1.1) 授權：

* 完全開源的程式碼庫，可用於自託管和客製化開發。
* 對於大規模商業使用或 SaaS 部署，需遵守 BSL 條款的特定限制。
* 官方也提供雲端託管的 SaaS 版本，進階功能可能會優先在此版本發布。

Twenty 非常適合尋求可客製化、開源 CRM 解決方案的技術團隊。對於大規模商業部署或再分發，建議團隊仔細審查 BSL 條款，並直接與 Twenty 確認相關條件。

## **#3: ERPNext ⭐️ 24.7k**

![ERPNext.png](https://static-docs.nocobase.com/3dcd2d82-4451-4e8a-b85b-54bf3d8b83aa-9fm9ri.png)

* GitHub 發布時間：2011 年
* 星星數：24.7k
* GitHub：[https://github.com/frappe/erpnext](https://github.com/frappe/erpnext)
* 網站：[https://erpnext.com](https://erpnext.com)

**概述**

ERPNext 由印度公司 Frappe 開發，於 2011 年開源發布，是全球最活躍且最受尊敬的開源 ERP 平台之一。它基於內部開發的 Frappe 框架（後端使用 Python、MariaDB/MySQL，前端使用 JavaScript/React），既提供即開即用的便利性，又具備強大的客製化能力。

該平台無縫整合了 ERP 和 CRM 的核心功能——涵蓋採購、庫存、銷售、會計、人力資源、專案管理、製造、支援、資產追蹤，甚至網站管理。它特別適合中小企業和快速成長的公司，能夠實現快速的數位轉型。其內建的低程式碼能力使團隊能夠透過直觀的拖放介面和配置工具，快速客製化資料模型和業務工作流程。

**主要優勢**

* **一體化模組：** 完全整合關鍵的 ERP 和 CRM 模組，提供銷售、庫存、財務、人力資源、CRM、專案等的統一管理。
* **高度可客製化：** 直觀的低程式碼環境允許業務用戶輕鬆建立自訂表單、欄位、權限、工作流程和自動化。
* **全球化與多公司支援：** 全面的多語言、多貨幣和多實體功能，非常適合跨國公司和分散式營運。
* **真正的開源：** 基於 GPL v3 授權提供完整原始碼，部署選項靈活，包括自託管和官方雲端服務，確保沒有供應商鎖定。

**適合對象**

* **快速成長的企業：** 旨在實現標準化營運和統一數位管理以應對快速成長的公司。
* **跨國營運：** 需要多語言和多貨幣支援以處理多樣化國際業務或多個子公司的企業。
* **尋求敏捷性的中小企業：** 希望快速部署並持續客製化 ERP，而不需過度依賴外部技術資源的企業。

**開源狀態**

ERPNext 在 GPL v3 授權下完全開源，提供對所有模組、底層技術和完整原始碼的無限制存取。使用者可以自由部署、客製化、整合和擴展 ERPNext，並獲得活躍社群的支援。

雖然 ERPNext 官方提供託管的雲端服務，但所有核心功能和程式碼仍可公開存取，用於自託管部署。

## **#4: Monica ⭐️ 22.7k**

![Monica.png](https://static-docs.nocobase.com/73d89234-f251-41c9-9506-fa03c90a64b6-02p6ij.png)

* GitHub 發布時間：2017 年
* 星星數：22.7k
* GitHub：[https://github.com/monicahq/monica](https://github.com/monicahq/monica)
* 網站：[https://www.monicahq.com](https://www.monicahq.com)

**概述**

Monica 是一個獨特的開源 CRM——它專注於個人關係管理，而非傳統的客戶管理。它使用 PHP（Laravel 框架）、MySQL 和 Vue.js 前端建構，幫助使用者追蹤和管理與朋友、家人、客戶和商業夥伴的互動。它易於部署、資源需求低，並強調易用性而非複雜的企業功能。

Monica 提供專注的功能，如聯絡人管理、任務、提醒、通話記錄，甚至個人喜好（如禮物偏好）等細節。與其說它提供全面的企業 CRM 功能，不如說 Monica 擅長為管理有意義的人際連結提供個人化的體驗。

**主要優勢**

* **個人關係管理：** 非常適合自由工作者、顧問、銷售人員，或任何需要有效管理大量個人或專業聯絡人的人。
* **使用者友善的介面：** 簡潔的設計和直觀的使用者體驗，緊密貼合日常使用場景。
* **簡單輕量的部署：** 清晰的部署流程、最少的依賴項和簡單的資源需求，使其非常適合希望快速上手的個人或小型團隊。
* **多語言與易於匯出：** 支援多種語言和輕鬆的資料匯出，非常適合全球使用和長期關係管理。
* **注重隱私的設計：** 優先考慮使用者隱私和資料所有權，非常適合關心資料控制和隱私的使用者。

**適合對象**

* **個人與自由工作者：** 希望系統化追蹤和長期管理重要聯絡人與關係的人。
* **小型顧問或服務團隊：** 需要基本 CRM 功能，但又不想陷入傳統企業級 ERP 系統複雜性的團隊。
* **注重隱私的使用者：** 偏好自託管解決方案以保持對資料的完全所有權，並將與 SaaS 替代方案相關的資料安全風險降至最低的使用者。

**開源狀態**

Monica 在 AGPL v3 授權下完全開源。所有功能和原始碼都在 GitHub 上公開，沒有受限制的商業功能。

* 支援自託管、自訂修改和廣泛的個人化設定。
* 提供官方 Docker 映像檔和清晰、易於遵循的部署指南。
* 官方提供託管的 SaaS 版本（"Monica Cloud"），但自託管使用者保留完整功能並可完全存取開源程式碼庫。

## **#5: Huly Platform ⭐️ 20.6k**

![Platform.png](https://static-docs.nocobase.com/90c69df3-0265-4ca7-b058-3dc8c2033516-p70lno.png)

* GitHub 發布時間：2022 年
* 星星數：20.6k
* GitHub：[https://github.com/hcengineering/platform](https://github.com/hcengineering/platform)
* 網站：[https://huly.io](https://huly.io)

**概述**

Platform 由 Human Colossus Foundation 開發，是一個次世代的開源資料與關係管理框架。與傳統的 ERP 或 CRM 解決方案不同，Platform 的核心在於資料主權、合規性和高度適應性的資料建模。它為企業和個人提供了一個可自我控制、可驗證的基礎設施，用於安全的資料協作。

Platform 使用 TypeScript、Node.js 和 PostgreSQL 建構，採用獨特的「關係驅動資料架構」，強調視覺化資料建模工具和進階權限管理。與其說它是一個獨立的 ERP 或 CRM，不如說 Platform 被設計為一個基礎設施，可以在其上建構客製化的 ERP、CRM 或其他複雜的業務系統，從而完全控制資料流和業務邏輯。

**主要優勢**

* **資料主權與合規性：** 非常適合有嚴格隱私和法規要求（如金融、醫療保健）的行業，強調安全、合規的資料處理。
* **靈活的資料建模：** 提供廣泛的工具用於元資料管理、關係建模和細粒度的資料權限，大大超越了傳統 ERP/CRM 模組的僵化性。
* **去中心化與可驗證框架：** 支援去中心化身份和分散式資料驗證，為新興的 Web3 和可信運算用例做好準備。
* **開放且可擴展的架構：** 便於與第三方解決方案整合，並允許團隊快速開發自訂應用程式和擴展。
* **透明、社群主導的開發：** 完全開放的治理模式，專案路線圖和持續開發過程公開透明，歡迎社群參與。

**適合對象**

* **高度監管行業：** 金融、醫療保健、政府或研究機構，需要對資料管理和權限進行嚴格控制和追蹤。
* **有複雜客製化需求的企業：** 希望定義自己獨特的資料結構和業務邏輯，而不是採用預先定義的 CRM/ERP 模型的企業。
* **面向未來的組織：** 探索去中心化資料治理、數位身份、Web3 基礎設施和新興資料主權趨勢的創新團隊。

**開源狀態**

Platform 在 Apache License 2.0 下完全開源，所有核心功能和原始碼均可完全存取。

**注意：** 作為基礎設施，Platform 需要一定的技術能力，特別是在資料建模和自訂應用程式開發方面。它最適合擁有成熟技術專長和內部開發能力的組織。
