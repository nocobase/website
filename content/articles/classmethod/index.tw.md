## 簡介

透過告別試算表，[Classmethod](https://classmethod.jp/) 利用 NocoBase 打造了一個結構清晰、存取權限靈活的員工資訊管理平台。

這個系統不僅簡化了他們的內部營運，也將使他們能夠為客戶提供更有效的技術解決方案。

## 資訊管理的挑戰：當 Excel 已不敷使用

隨著 Classmethod 業務的成長，其團隊結構也變得越來越複雜——目前日本總部約有 500 名員工，集團總計超過 850 人。員工組成不僅包含全職人員，還有兼職人員、斜槓合作夥伴、約聘人員及合作夥伴，遍佈日本國內外。

伴隨著這樣的成長，Classmethod 開始尋找更好的方式來管理其組織：

1.  如何用單一、統一的標準來管理不同國家和不同僱用類型的員工？
2.  如何自動化入職、離職或內部調動等流程——涵蓋資料輸入、存取權限和通知？
3.  公司如何集中追蹤每個人的認證、專業背景和培訓記錄，以支援更明智的工作指派、人才發展和招募？
4.  系統化管理如何不僅提升營運效率，還能為初階工程師提供實際的營運與開發經驗——同時控制系統成本？

現在，想像一下試圖僅用 Excel 或 Google Sheets 來處理這一切。很容易就能看出問題會迅速累積。事實上，這些正是 Classmethod 團隊親身經歷的痛點：

| **使用 Excel 和 Google Sheets 管理**                              | 關鍵挑戰                                                                                                           |
| ---------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------ |
| ⚠️ 軟體授權記錄保存在 Excel 中                                       | ❌ 授權指派經常不準確——未使用的帳戶仍佔用有效授權                                                                  |
| ⚠️ 各部門各自管理自己的 Google Sheet                              | ❌ 稽核變得緩慢，對使用者和 IT 部門都是負擔                                                               |
| ⚠️ 資料存取權限記錄在獨立的試算表中     | ❌ 資料定義不一致，導致資訊共享與管理困難，且權限容易設定錯誤 |
| ⚠️ 複雜資訊需手動使用 VLOOKUP 進行交叉比對 | ❌ 跨部門查找資訊困難，拖慢工作流程                                          |
| ⚠️ 每次入職、調動或離職都需手動更新  | ❌ 入職流程可能耗時長達 10 個工作天，為新進員工帶來不佳體驗                                          |

最重要的是，隨著團隊成長和流程變得更加複雜，這些問題只會惡化。缺乏集中且自動化的系統，導致管理團隊的負擔增加——而整個組織也因反應變慢和資料可靠性降低而受到影響。

這就是為什麼 Classmethod 開始尋找更具系統性且靈活的解決方案：一個能夠集中管理資料、支援業務特定配置，並能隨著時間實現自動化的方案。

## 系統選型過程

當 Classmethod 決定告別試算表時，團隊評估了一系列工具，包括 Airtable 和 NocoDB。這些解決方案以其易用性、直觀的介面以及某種程度的可擴展性而脫穎而出。然而，最終團隊選擇 NocoBase 的原因很明確：它提供了其他工具無法比擬的靈活性、控制力以及與 Classmethod 需求的契合度。

以下是他們決策中最關鍵的因素：

**1. 靈活的開源部署**

作為一家技術驅動的公司，Classmethod 傾向於在自己的 AWS 環境中運行關鍵系統，以滿足安全與合規要求。NocoBase 是完全開源的，可以部署在任何地方，不受 SaaS 限制。

**2. 無需按用戶收費——非常適合「瀏覽者多、編輯者少」的情境**

雖然只有少數人負責輸入資料，但許多部門都需要存取權限。若按用戶數計價，成本會迅速攀升。NocoBase 的開源模式完全避免了這個問題。

**3. 細粒度的存取控制**

NocoBase 支援針對不同使用者設定記錄級和欄位級的可見性權限——這對於在員工管理系統中實現安全、分層的資料存取至關重要。

**4. UI 與資料結構分離，可自訂介面**

不同團隊需要查看不同的資訊。使用 NocoBase，UI 和資料模型可以獨立配置，使得建立靈活、客製化的介面變得容易。

**5. 內建工作流程引擎——無需外部自動化工具**

像是為新進員工自動建立帳戶或撤銷臨時權限等業務邏輯，可以完全透過內建的工作流程來處理，無需第三方自動化工具或自訂腳本。

**6. 自動生成 API，便於整合**

NocoBase 中的每個新資料集合都會自動生成 API，這使得工程團隊能夠更輕鬆地自動化腳本和同步資料。

簡而言之，NocoBase 的功能列表可能不是最長的，但它精準地提供了 Classmethod 最需要的東西：安全性、自主性、靈活性，以及對開發和營運的完全掌控。

## NocoBase 實戰：初步成效

計畫確定後，Classmethod 迅速部署了 NocoBase 並建立了資料模型。工程團隊在他們自己的 AWS 環境中部署了 NocoBase，並設置了反映組織結構的核心資料集合——涵蓋員工、部門、軟體授權、公司資產、系統帳戶（如 Slack 和 GitHub）以及認證。業務用戶也可以根據需要更新欄位和調整 UI 佈局。

初始資料透過 CSV 導入，後續更新則使用 Python 腳本和 NocoBase 自動生成的 API 來自動化。透過關聯欄位連結所有資料，團隊實現了真正結構化的資訊管理。

上線後，新系統帶來了顯著的效率提升：

*   資料不再分散於各部門的試算表中，大大減少了同步問題和版本衝突
*   SaaS 授權管理實現自動化，使 IT 稽核效率大幅提升
*   像是撤銷臨時權限、確認設備歸屬以及發送到期提醒等任務，越來越趨向自動化
*   業務團隊可以自行調整欄位和介面——不再需要等待工程師，維護工作也變得更加輕量
*   人才資料（認證、培訓、經驗）現在已結構化，使得管理內部調動和招募更加容易

## 持續改進：將內部成功轉化為客戶價值

Classmethod 現在已將其日常員工資訊管理完全遷移到 NocoBase，並正在推動自動化和整合，例如：

*   當新增或更新資料時觸發審批工作流程
*   透過 Slack 向其他團隊發送通知
*   將資料匯出到外部系統（如 Google Sheets 或整合平台）
*   在授權到期前自動請求續約
*   確認資產歸屬（如筆記型電腦、安全徽章等）
*   自動撤銷臨時權限
*   透過自動化工作流程發送授權到期提醒
*   透過其 MCP 伺服器與大型語言模型整合

該團隊也致力於更好地視覺化員工資料——例如認證、工作經歷和培訓記錄——以改善招募、人才發展和內部知識共享。

藉由 NocoBase，Classmethod 不僅集中並自動化了其人力資源管理，同時也累積了寶貴的系統開發與營運實戰經驗。

作為一家技術驅動的 IT 服務公司，Classmethod 致力於將實務經驗轉化為客戶的實際價值。現在，他們正將這些經過驗證的做法應用於客戶專案——幫助更多企業在當今複雜的環境中建立靈活、可控的系統。
