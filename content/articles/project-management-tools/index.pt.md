Imagine que você é um gerente de projetos. Toda manhã, ao se acomodar em sua mesa com seu café na mão, as telas do seu telefone e computador já estão cheias de uma infinidade de tarefas inacabadas, atualizações de projetos e mensagens da equipe.

E-mails passam voando, notificações de atribuição de tarefas não param de aparecer, cada uma exigindo que você tome decisões rápidas. Enquanto isso, os membros da equipe estão enviando mensagens para você sobre atualizações de progresso, discutindo problemas, e alguns até trazem novos desafios e solicitações de mudança diretamente para as reuniões.

Você precisa urgentemente de uma solução para atender às seguintes necessidades específicas:

1. **Simplificar a Atribuição de Tarefas e o Gerenciamento de Prioridades**: Atribuir tarefas facilmente e definir prioridades para garantir que os membros da equipe entendam suas responsabilidades, evitando omissões e trabalho duplicado.
2. **Acompanhamento de Progresso e Relatórios em Tempo Real**: Acompanhar o progresso do projeto em tempo real, gerar relatórios e análises de dados automaticamente para ajudar a identificar problemas prontamente e tomar decisões baseadas em dados.
3. **Melhorar a Comunicação e Colaboração da Equipe**: Aprimorar os processos de comunicação interna com uma plataforma centralizada para transmitir informações rapidamente, reduzindo mal-entendidos e atrasos.
4. **Integração e Gerenciamento de Documentos**: Integrar-se com aplicativos e sistemas de trabalho existentes, armazenar e gerenciar centralmente todos os documentos e materiais relacionados ao projeto para fácil acesso e compartilhamento.
5. **Personalização Flexível de Fluxo de Trabalho**: Suportar a personalização de várias metodologias e fluxos de trabalho de gerenciamento de projetos para se adaptar a diferentes necessidades de projeto e requisitos da equipe.

Ao delinear esses requisitos, você pode pensar que um sistema de gerenciamento de projetos capaz de atender a demandas tão complexas exigiria tempo e recursos significativos, potencialmente precisando de uma equipe de desenvolvimento dedicada para desenvolvimento personalizado, o que sem dúvida sobrecarregaria o orçamento.

## 2. **Taiga**

![Taiga.png](https://static-docs.nocobase.com/285fc99825c4ce24dd1435e0d2029a5a.png)

**Experiência do Usuário**

[Taiga](https://taiga.io/) é uma ferramenta de código aberto que encontrei ao pesquisar soluções de gerenciamento de projetos ágeis. É particularmente adequada para desenvolvimento ágil, oferecendo recursos como quadros de tarefas, gerenciamento de iterações e rastreamento de requisitos para ajudar as equipes a gerenciar o progresso do projeto de forma eficiente.

💡 Leia Mais: [Explorando RAD: 5 Melhores Casos de Aplicação](https://www.nocobase.com/en/blog/rapid-application-development-best-application-cases)

**Vantagens**

* **Suporte Ágil:** Fornece funcionalidades necessárias para o desenvolvimento ágil, como gerenciamento de iterações, quadros de tarefas e listas de afazeres.
* **Visualizações Abrangentes do Projeto:** Oferece várias visualizações para gerenciar projetos, incluindo visualização de quadro, visualização de iteração e listas de afazeres.
* **Relatórios e Análises:** Fornece ferramentas detalhadas de relatórios e análises para monitorar o progresso do projeto, identificar possíveis problemas e tomar decisões baseadas em dados.

**Casos de Uso**

O Taiga é ideal para cenários de gerenciamento de projetos que exigem desenvolvimento ágil e colaboração eficiente, incluindo projetos que usam métodos Scrum e Kanban, colaboração de equipes multifuncionais, desenvolvimento rápido de produtos com iterações e gerenciamento de múltiplas tarefas.

**Início Rápido**

Antes da instalação, certifique-se de ter o seguinte software e esteja familiarizado com as tecnologias relevantes:

* `docker`: Versão >= 19.03.0+
* Docker, docker compose e Docker Repository

1. Obtenha o Repositório

```Plain
$ cd taiga-docker/
$ git checkout stable
```

2. Configuração

As configurações básicas do Taiga são expostas no arquivo `.env`. Revise e modifique este arquivo conforme necessário para evitar valores padrão.

a. Abra o arquivo `.env` e faça as alterações necessárias.

b. Para personalizações adicionais ou configurações avançadas, edite os arquivos `docker-compose.yml` e `docker-compose-inits.yml`.

## 3. OpenProject

![OpenProject.png](https://static-docs.nocobase.com/e3b8279c7d043018f4399c7b5c2aa8fb.png)

**Experiência do Usuário**

Ao buscar uma solução abrangente de gerenciamento de projetos, o [OpenProject](https://www.openproject.org/) chamou minha atenção. Ele oferece um rico conjunto de módulos, incluindo atribuição de tarefas, acompanhamento de progresso e gerenciamento de documentos, abordando efetivamente vários problemas de gerenciamento de projetos.

**Vantagens**

* **Ferramentas Abrangentes:** Fornece ferramentas detalhadas de planejamento de projetos, incluindo gráficos de Gantt, marcos e Estruturas Analíticas de Projeto (EAP).
* **Gerenciamento Flexível de Tarefas:** Permite criar, atribuir, definir prioridades, acompanhar o progresso e gerenciar tarefas através de visualizações de quadro e listas de tarefas.
* **Colaboração da Equipe:** Facilita a colaboração entre os membros da equipe, incluindo discussões, compartilhamento de arquivos e comunicação em tempo real.

**Casos de Uso**

O OpenProject é adequado para cenários que exigem gerenciamento eficiente de projetos e colaboração diversificada, incluindo planejamento e acompanhamento detalhados de projetos, colaboração de equipes interdepartamentais, decomposição e acompanhamento de tarefas complexas, gerenciamento de recursos e controle de riscos.

**Início Rápido**

A maneira mais rápida de iniciar e executar uma instância do OpenProject é usando o seguinte comando:

```Bash
docker run -it -p 8080:80 \-e OPENPROJECT_SECRET_KEY_BASE=secret \-e OPENPROJECT_HOST__NAME=localhost:8080 \-e OPENPROJECT_HTTPS=false \-e OPENPROJECT_DEFAULT__LANGUAGE=en \
  openproject/openproject:14
```

Notas de Configuração:

* `-p 8080:80` vincula a porta 80 do contêiner à porta 8080 na máquina que executa o docker.
* `OPENPROJECT_SECRET_KEY_BASE` define a chave secreta base para Rails. Use um valor pseudoaleatório para isso e trate-o como uma senha.
* `OPENPROJECT_HOST__NAME` define o nome do host da aplicação. Este valor é usado para gerar formulários e links em e-mails e precisa corresponder ao nome do host da solicitação externa (O valor que os usuários veem em seus navegadores).
* `OPENPROJECT_HTTPS=false` desabilita o modo HTTPS ativado por padrão do OpenProject para que você possa acessar a instância apenas via HTTP. Para todos os sistemas de produção, recomendamos fortemente não definir isso como falso e, em vez disso, configurar uma terminação TLS/SSL adequada em seu servidor web externo.
* `OPENPROJECT_DEFAULT__LANGUAGE` faz duas coisas. Controla, para a primeira instalação, em qual idioma os dados básicos (como tipos, nomes de status, etc.) e dados de demonstração são criados. Também define o idioma de fallback padrão para novos usuários.

Isso levará algum tempo na primeira vez que você iniciá-lo, mas após alguns minutos você verá uma mensagem de sucesso indicando a senha de administração padrão (login: `admin`, senha: `admin`).

Você pode então abrir um navegador e acessar sua nova instalação do OpenProject em `http://localhost:8080`. Fácil!

Para parar o contêiner, basta pressionar CTRL-C.

Documentação Detalhada: [OpenProject em contêiner all-in-one Docker](https://www.openproject.org/docs/installation-and-operations/installation/docker/#quick-start)

## 4. **Kanboard**

![Kanboard.png](https://static-docs.nocobase.com/6e351b3a74f88ed3cf9646204fe73e88.png)

**Experiência do Usuário**

[Kanboard](https://kanboard.org/) é uma ferramenta de gerenciamento de projetos de código aberto focada na metodologia Kanban. Sua interface simples e ênfase na visualização de tarefas tornam o acompanhamento do progresso do projeto direto.

💡 Leia Mais: [Top 6 Ferramentas de Criação de Painéis de Aplicativos Principais](https://www.nocobase.com/en/blog/core-app-dashboard-building-tools)

**Vantagens**

* **Interface Intuitiva:** Fornece uma interface de quadro Kanban clara e fácil de usar, permitindo atualizações rápidas no status das tarefas através de operações de arrastar e soltar.
* **Fluxos de Trabalho Flexíveis:** Personalize fluxos de trabalho de acordo com as necessidades do projeto, definindo diferentes status de tarefas e regras de transição para se adequar a vários tipos de projeto e hábitos da equipe.
* **Privacidade e Controle de Dados:** Implantado em servidores locais para evitar problemas de privacidade de dados associados a serviços de terceiros.

**Casos de Uso**

O Kanboard é adequado para cenários que exigem gerenciamento simplificado de tarefas e configurações flexíveis de fluxo de trabalho, incluindo acompanhamento do progresso de tarefas com uma visualização de quadro intuitiva, ajuste rápido de status de tarefas, colaboração da equipe e gerenciamento de prioridades.

**Início Rápido**

Aqui está um guia de instalação simplificado para o Kanboard em sistemas Debian:

1. **Atualize o Sistema:**

```Bash
sudo apt update
sudo apt upgrade -y
```

2. **Instale os Pacotes Necessários:**

```Bash
sudo apt install -y apache2 mariadb-server php php-mysql php-gd php-xml php-mbstring git
```

3. **Baixe o Kanboard:**

```Bash
cd /var/www/html
sudo git clone https://github.com/kanboard/kanboard.git
sudo mv kanboard kanboard-app
```

4. **Configure as Permissões:**

```Bash
sudo chown -R www-data:www-data /var/www/html/kanboard-app 
sudo chmod -R 755 /var/www/html/kanboard-app
```

5. **Configure o Apache:**

   a. Crie um novo arquivo de configuração do Apache `/etc/apache2/sites-available/kanboard.conf` com o seguinte conteúdo:

   ```Apache
   <VirtualHost *:80>
       ServerAdmin webmaster@localhost
       DocumentRoot /var/www/html/kanboard-app
       ErrorLog ${APACHE_LOG_DIR}/error.log
       CustomLog ${APACHE_LOG_DIR}/access.log combined

       <Directory /var/www/html/kanboard-app>
           AllowOverride All
           Require all granted
       </Directory>
   </VirtualHost>
   ```

   b. Habilite a nova configuração e reinicie o Apache:

   ```Bash
   sudo a2ensite kanboard
   sudo a2enmod rewrite
   sudo systemctl restart apache2
   ```
6. **Configure o Banco de Dados:**

   a. Faça login no MariaDB:

   ```Bash
   sudo mysql -u root
   ```

   b. Crie o banco de dados e o usuário, e conceda permissões:

   ```Bash
   CREATE DATABASE kanboard;
   CREATE USER 'kanboarduser'@'localhost' IDENTIFIED BY 'password';
   GRANT ALL PRIVILEGES ON kanboard.* TO 'kanboarduser'@'localhost';
   FLUSH PRIVILEGES;
   EXIT;
   ```
7. **Configure o Kanboard:**

   a. Abra seu navegador e vá para `http://your_server_ip/kanboard-app`.

   b. Siga as instruções na página para concluir a instalação, inserindo informações do banco de dados e outras configurações.

Documentação Detalhada: [Documentação do Kanboard](https://docs.kanboard.org/v1/admin/installation/)

## 5. **Focalboard**

![Focalboard.png](https://static-docs.nocobase.com/076e31500670995a5c8a91b7ae0cc2e4.png)

**Experiência do Usuário**

[Focalboard](https://www.focalboard.com/) é uma ferramenta de gerenciamento de projetos de código aberto emergente que oferece quadros de tarefas, visualização de calendário e rastreamento de tarefas, com capacidades de integração com outras ferramentas.

**Vantagens**

* **Recursos de Personalização:** Permite a personalização de campos de tarefas e fluxos de trabalho, suportando a criação de várias visualizações e quadros.
* **Múltiplas Visualizações:** Suporta várias opções de visualização, incluindo quadros Kanban, visualização de lista e visualização de calendário.
* **Colaboração em Tempo Real:** Facilita a colaboração em tempo real entre os membros da equipe, exibindo atribuições de tarefas, status de progresso e conteúdo de discussão.

**Casos de Uso**

O Focalboard é adequado para cenários de gerenciamento de projetos que exigem fluxos de trabalho personalizáveis e gerenciamento flexível de tarefas, incluindo colaboração em tempo real, opções de visualização ricas (visualizações Kanban, lista e calendário), proteção de privacidade de dados e diversos recursos de rastreamento de tarefas.

**Início Rápido**

Use o único comando [Docker](https://www.docker.com/) para baixar e executar a versão mais recente do Focalboard Personal Server:

```Plain
docker run -it -p 80:8000 mattermost/focalboard
```

Em seguida, abra seu navegador e vá para [http://localhost](http://localhost/).

Para especificar um número de porta:

```Plain
docker run -it -p <port>:8000 mattermost/focalboard
```

Documentação Detalhada: [Personal Server (Docker)](https://www.focalboard.com/download/personal-edition/docker/)
