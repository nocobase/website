## はじめに

[Astrolab](https://astrolab.co/) は、ヨーロッパ、アフリカ、中東における NocoBase のパートナーの一社です。技術戦略、インテリジェントオートメーション、AI、デジタルソリューションの導入支援を専門とするテクノロジーサービス企業です。

![Astrolab-jwcfkt.png](https://static-docs.nocobase.com/Astrolab-jwcfkt.png)

Astrolab は NocoBase を基盤に、フランスのある地域政府向けに3つの本番運用レベルのデジタル公共サービスプラットフォームを構築しました。

これらは、それぞれ独立した単発のツールではありません。同じアーキテクチャの上に構築された一連の公共サービスシステムです。紙の申請書、スプレッドシート、メールでのやり取りに依存していた業務プロセスを、柔軟に管理できるデジタルサービスへ移行することを支援しました。

## 課題

このフランスの地域政府は、数百万人の住民に関わる環境政策、教育、経済発展、交通、国土計画を担っています。一方で、長年の運用課題も抱えていました。**複数の重要な行政プロセスが、いまだに紙のフォーム、スプレッドシート、ばらばらのメールベースのワークフローで処理されていたのです。**

なかでも、以下の3つの公共サービスは優先度が高く、特にデジタル化が求められていました。

**GIE 環境影響評価**
この地域の経済発展支援を申請する企業は、多面的な環境自己評価を行う必要があります。評価項目には、炭素排出量、エネルギー使用量、水資源の消費、廃棄物管理、土地利用、生態系に関する指標などが含まれます。

従来のプロセスは、手作業による審査と重複入力に大きく依存していました。その結果、承認の遅れ、評価基準のばらつきが発生し、管理者が進捗をリアルタイムで把握することも困難でした。

**SID 公用車両共有管理**
この地域政府では、複数の部門が車両を共同で利用しています。しかし従来は、手作業での予約とスプレッドシートへの記録が中心でした。そのため、予約の重複、車両利用率の低さ、走行距離や燃料消費データの不透明さといった問題が起きやすい状況でした。

**EIT 年次寄宿学校調査**
地域の教育部門は、毎年190以上の公立・私立機関を対象に調査を行う必要があります。以前はメールと紙の資料でデータを集めていたため、回答率が低く、集計作業の負担も大きく、レポート作成にも長い時間がかかっていました。データの完全性や一貫性を保つことも難しく、地域の教育計画にも影響していました。

## Astrolab のソリューション

Astrolab は、統一されたフレームワークのもとで、3つの本番運用レベルのデジタル公共サービスプラットフォームを提供しました。いずれのプラットフォームも NocoBase をベースに設計・構築されており、NocoBase は次のような基盤機能を担っています。

* 統一された認証とロール管理
* 共通のワークフローオーケストレーション層
* 集中管理された監査ログ基盤
* 通知・リマインダーエンジン
* 動的フォームとデータモデル管理
* API 統合フレームワーク
* 業務チームがすばやく改善を重ねられる、モジュール型のローコード管理機能

このアプローチにより、**各システムで共通機能を何度も作り直す必要がなくなりました**。開発・導入チームは NocoBase の既存機能を活用しながら、業務プロセス、データルール、権限設計、システム連携といった本質的な部分に集中できました。

このアーキテクチャの狙いは、公共部門に求められるガバナンス、セキュリティ、拡張性、保守性を満たしながら、導入期間を短縮することです。同時に、システム公開後も業務チーム自身がフォーム、プロセス、管理ルールを継続的に調整できるようにしています。

また、提供されたすべてのプラットフォームは、次の基準に準拠しています。

* フランス **DINUM** のデジタルサービス基準
* **GDPR**（一般データ保護規則）の要件
* フランス **RGAA** アクセシビリティ基準（WCAG に相当）
* **ANSSI**（フランス国家サイバーセキュリティ庁）が求める公共部門システム向けセキュリティベースライン
* すべてのデータをフランス国内の**主権インフラ**上でホスティング

### プラットフォーム1：GIE 環境評価

GIE 環境評価プラットフォームは、企業が経済発展支援を申請する際の環境自己評価プロセスを処理するためのシステムです。

Astrolab は NocoBase を使い、**動的な調査・審査システム** を構築しました。企業がシステムにアクセスすると、質問票は企業規模や業種に応じて自動的に調整されます。また、SIRET コード（フランスの企業登録番号）を使って、国家の公共 API から企業情報を取得できるため、重複入力や手作業による確認を減らせます。

評価が提出されると、システムはユーザーのロールに応じて自動的に審査ルートへ進み、地域の補助金承認システムとも連携します。プロセス全体には改ざんできない監査履歴が残り、企業側は入力途中の状態を一時保存して、後から再開することもできます。

業務チームにとって特に重要なのは、運用開始後の保守性です。NocoBase のローコード管理画面を使えば、フォーム項目、調査フロー、審査ルールを継続的に調整できます。変更のたびに再開発する必要はありません。

![GIE.png](https://static-docs.nocobase.com/image-b8ijcx.png)

### プラットフォーム2：SID 車両共有

SID 車両共有プラットフォームは、地域政府の複数部門が共同利用する車両を管理するためのシステムです。

Astrolab は、スプレッドシートと手作業の連絡に依存していた車両予約プロセスを、オンライン予約・承認システムへ移行しました。運転者はシステム上で車両を確認して予約を申請でき、管理者はその申請を承認できます。システムは時間の重複を自動で検出し、二重予約を防ぎます。

プラットフォームでは、走行距離、燃料消費、部門ごとの利用状況も記録できます。部門別のレポートダッシュボードも用意されており、管理者は Excel や PDF のレポートを出力できます。日常的な予約や管理は、モバイル端末からも行えます。

このプラットフォームは、NocoBase のワークフロー、権限管理、データ管理機能を基盤に構築されています。車両利用ルール、承認フロー、集計基準が変わった場合でも、管理者は既存システム内ですばやく調整できます。

### プラットフォーム3：EIT 調査プラットフォーム

EIT 調査プラットフォームは、地域の教育部門が年次の寄宿学校データ収集を行うためのシステムです。

Astrolab は NocoBase を使い、このプロセスをオンライン調査システムへ移行しました。

プラットフォームでは、公立機関と私立機関に対して異なる調査フローを設計できます。スマートな分岐ロジックと過去データの事前入力にも対応しているため、各機関は重複入力を減らせます。管理者はリアルタイムの進捗ダッシュボードで提出状況を確認でき、集中型のデータ検証プロセスによってデータ品質も高められます。

調査票やレポート構造は、毎年変わる可能性があります。NocoBase のノーコード・ローコードアーキテクチャにより、業務チームはシステム公開後も調査票、フィールド、レポート構造を継続的に調整できます。そのため、毎年長い再開発サイクルに入る必要がありません。

## 成果

3つのプラットフォームが本番運用に入った後、顧客はプロセス処理の効率、データ品質、運用の可視性、コンプライアンスガバナンスの面で大きな改善を実感しました。

最も分かりやすい変化は、環境評価プロセスで見られました。以前は手作業の審査に数日かかっていましたが、現在は**24時間以内**に短縮されています。公共部門の登録台帳やオープンデータ**システムとの12件以上のリアルタイム連携**により、プラットフォームは**企業プロファイル情報の95%以上を自動入力**できるようになりました。これにより、重複入力や手作業での確認が減り、評価結果も財政支援の承認プロセスへ自動的にルーティングされます。

車両共有管理では、オンライン予約、自動競合検出、部門別の運用ビューによって、より直接的な管理効果が生まれました。**車両予約の競合は約80%減少し、車両利用率は約25%向上**しました。各部門の走行距離や燃料消費も継続的に追跡できるようになっています。

年次寄宿学校調査の効率も大きく改善しました。プラットフォーム公開後の最初の運用サイクルでは、**調査票の回収率が90%以上に向上し、データ統合にかかる時間は数週間から48時間以内へ短縮**されました。より完全で一貫性のあるデータは、その後の教育計画にとっても信頼できる基盤になっています。

効率面だけでなく、3つのプラットフォームは公共部門システムに必要なガバナンス機能も強化しました。**すべてのワークフローに完全な追跡記録があり、プラットフォームは集中型の監査ログを提供し、年次の侵入テストにも合格しています**。プロジェクト運用期間中、既知のデータ漏えいは発生しておらず、プラットフォームのコンプライアンスも、顧客のガバナンスチームによる GDPR 要件の確認を通過しています。

安定性の面でも、3つのプラットフォームは24か月の運用期間中、実測のサービス可用性が99.8%を超えました。継続的なサポートのもと、Astrolab は四半期ごとの機能リリースを維持し、3つのサービスはいずれも本番環境で安定稼働を続けています。

## 技術アーキテクチャ

Astrolab は、3つのプラットフォームの共通基盤として NocoBase を採用しました。NocoBase は、データモデル、動的フォーム、権限制御、ワークフロー、監査ログ、管理画面などの共通機能を担っています。

その上で、Astrolab は React.js、NestJS、PostgreSQL、PostGIS、Power BI、カスタムレポートエンジン、政府 API 連携、Docker、CI/CD、プライベートクラウドデプロイを組み合わせ、公共部門の本番環境に対応するシステムを完成させました。

このアーキテクチャは、データ主権、RGPD/GDPR 準拠、RBAC、AES-256 暗号化、監査ログ、改ざんできない監査記録、ゼロダウンタイムデプロイなどの要件もカバーしており、3つのプラットフォームの長期的な安定運用を支えています。

導入プロセスでは、Astrolab は AI を活用した開発方法も試しました。たとえば Claude を使い、一部のカスタムプラグインコードの生成を支援しています。NocoBase 2.1 で AI 支援開発機能が強化されたことで、導入チームは AI Agent、NocoBase ドキュメント、プラットフォームの拡張メカニズムをより自然に組み合わせられるようになりました。これにより、プラグイン開発、JS Block の構築、複雑な業務ロジックの実装を進めやすくなっています。

この点からも、NocoBase は AI が構築と長期保守に関わる業務システムの基盤として、より適した選択肢になっています。

## まとめ

Astrolab は NocoBase を活用し、複数のシステムで共通する基盤機能を再利用しました。その上で、それぞれの業務シーンに合わせて拡張を行い、長期運用できる本番システムを提供しました。

これは、AI 時代において特に注目すべきシステム開発のあり方でもあります。

AI はコード生成や開発を加速できます。しかし、本当に使える業務システムは、一度生成して終わりの開発だけでは成立しません。安定したデータモデル、明確な権限設計、追跡可能なプロセス、監査できる履歴、そして継続的に調整できるシステムアーキテクチャが必要です。

NocoBase の価値は、まさにここにあります。**AI も導入チームもゼロから始める必要はありません。業務システムに必要な基盤機能をすでに備えたプラットフォーム上で、実際のアプリケーションを構築し、拡張し、長期的に保守できます。**
