![APIJSON.png](https://static-docs.nocobase.com/f9ffd26987c00fa0283a9de3f5faba95.png)

APIJSON es un protocolo de comunicación basado en JSON y una biblioteca ORM diseñada para API. Permite a los desarrolladores generar automáticamente interfaces backend y código de solicitud front-end definiendo objetos JSON en lugar de escribir código backend. APIJSON es particularmente adecuado para escenarios que requieren un desarrollo rápido de API, reduciendo la sobrecarga de comunicación y el trabajo de documentación de API, disminuyendo así significativamente los costos de desarrollo.

**Características principales**

* **Operaciones CRUD automatizadas**: APIJSON proporciona potentes funciones CRUD que admiten consultas complejas mediante una configuración JSON simple, incluyendo consultas JOIN y condiciones de filtrado complejas.
* **Seguridad y flexibilidad**: Mecanismos integrados de validación automática de permisos y protección contra inyección SQL para garantizar la seguridad de la API. APIJSON también admite la gestión de versiones, por lo que las iteraciones de API no afectan a los sistemas existentes.
* **Manejo eficiente de datos**: Admite la obtención de múltiples recursos y formas de datos en una sola solicitud, reduciendo la necesidad de múltiples solicitudes y optimizando la eficiencia de la interacción de datos.

**Casos de uso**

* **Desarrollo rápido**: Adecuado para startups y PYMEs que necesitan crear prototipos y desarrollar aplicaciones rápidamente, especialmente cuando los proyectos requieren iteración rápida y cambios frecuentes en la API.
* **Herramientas internas**: Configura rápidamente interfaces de gestión backend para sistemas de gestión empresarial y sistemas de monitoreo.
* **Educación y aprendizaje**: Debido a su facilidad de uso, APIJSON también es adecuado para entornos educativos para ayudar a estudiantes y principiantes a comprender la construcción y el uso de API.

Library es un ejemplo práctico destinado a ayudar a las personas a comprender cómo aplicar el Diseño Guiado por el Dominio (DDD) en el mundo real. Library se centra en un sistema de gestión de bibliotecas, similar al que utilizan las bibliotecas públicas para gestionar libros, lectores, préstamos y devoluciones.

El proyecto utiliza una arquitectura monolítica modular, con cada contexto delimitado asignado a un paquete separado, demostrando procesos de negocio complejos a través de una rica lógica de dominio y operaciones CRUD. Este proyecto no solo muestra cómo construir lógica de negocio compleja, sino que también cierra la brecha entre la teoría y la práctica con código real.

**Características principales**

* **Modelo de dominio rico**: El proyecto Library utiliza un modelo de dominio para manejar la lógica de negocio compleja, como el proceso de préstamo de libros, utilizando una arquitectura hexagonal para separar la lógica de dominio y aplicación.
* **Aplicación CRUD flexible**: Para la gestión del catálogo de libros, el proyecto aplica una arquitectura CRUD para manejar una lógica de dominio relativamente simple.
* **Consistencia basada en eventos**: Library utiliza eventos para gestionar la comunicación entre agregados, admitiendo opciones de consistencia inmediata y eventual, demostrando cómo lograr estas funciones a través de interfaces de eventos de dominio.

**Casos de uso**

* **Sistemas de gestión de bibliotecas**: Gestiona procesos como préstamo, devolución y reserva de libros, adecuado para sistemas que manejan lógica de negocio y reglas complejas.
* **Educación e investigación**: Como ejemplo de implementación de DDD, adecuado para investigación académica y formación educativa, ayudando a comprender los detalles y desafíos del diseño guiado por el dominio en aplicaciones prácticas.
* **Diseño de arquitectura de software**: Proporciona experiencia práctica en diseño de arquitectura para desarrolladores de software, especialmente en el uso de DDD y arquitectura basada en eventos.

* **Independencia de servicio y base de datos**: Se puede usar con cualquier base de datos o servicio, proporcionando controladores CRUD altamente extensibles.
* **Opciones de configuración ricas**: Admite configuración global y validación, proporcionando decoradores auxiliares adicionales para simplificar el proceso de desarrollo.

**Casos de uso**

* **Aplicaciones de nivel empresarial**: Adecuado para escenarios que requieren un desarrollo rápido de servicios backend empresariales, especialmente proyectos que necesitan cumplir estrictamente con los estándares RESTful.

* **Funcionalidad rica**: Admite operaciones CRUD, generación automática de documentación (OpenAPI), consultas complejas y gestión de permisos de datos.

* **Proyectos pequeños**: Adecuado para proyectos pequeños o medianos que necesitan una solución backend rápida y sencilla.

Estas herramientas se adaptan a una variedad de necesidades de proyectos, desde proyectos a pequeña escala hasta grandes aplicaciones empresariales y, a través de su naturaleza de código abierto, ofrecen una alta personalización y extensibilidad.
