## FAQ

## 1. O que você deve considerar ao escolher um Assistente de IA de código aberto?

Ao escolher um Assistente de IA de código aberto, você pode se concentrar em cinco aspectos: se o projeto é mantido ativamente, se suporta seu caso de uso, se pode se conectar a ferramentas ou sistemas existentes, se suporta auto-hospedagem ou implantação privada e se possui documentação clara e uma comunidade ativa.

Usuários individuais podem priorizar facilidade de uso, suporte a modelos, chamada de ferramentas locais e experiência no desktop. Usuários empresariais devem se concentrar mais em permissões, estruturas de dados, automação de fluxo de trabalho, integração de sistemas e recursos de manutenção de longo prazo.

## 2. Se eu quiser construir sistemas de negócios internos para uma empresa, qual projeto de Assistente de IA devo escolher?

Se seu objetivo é construir sistemas de negócios internos empresariais, como CRM, sistemas de aprovação, sistemas de tickets, gerenciamento de projetos, gerenciamento de inventário e backends de operações, você pode olhar primeiro para o **NocoBase**.

O NocoBase é mais adequado para suportar dados de negócios empresariais, permissões, fluxos de trabalho e configuração de página. Ele não só pode ajudar a construir sistemas com IA, mas também continuar ajustando campos, páginas, permissões e fluxos de trabalho através de uma interface no-code após a geração. Para sistemas internos empresariais que exigem manutenção de longo prazo, este tipo de plataforma estruturada é mais confiável.

## 3. Se eu só preciso de um Assistente de IA de desktop para uso pessoal, qual devo escolher?

Se você o usa principalmente pessoalmente e deseja usar vários modelos, processar arquivos, chamar ferramentas e organizar informações a partir de um único aplicativo de desktop, você pode olhar primeiro para o **DeepChat** ou **Everywhere**.

O DeepChat é mais adequado para trazer vários modelos, MCP, Habilidades e recursos de controle remoto para um único aplicativo de desktop. O Everywhere enfatiza a consciência de contexto de tela e compreensão contextual, tornando-o adequado para ler, traduzir, solucionar problemas e resumir páginas da web ou documentos.

## 4. Se eu sou um desenvolvedor e quero usar um Assistente de IA dentro de ferramentas de desenvolvimento, qual devo escolher?

Se você é um desenvolvedor no ecossistema Apple e usa principalmente o Xcode para construir projetos iOS, macOS, Swift ou Objective-C, você pode olhar para o **GitHub Copilot for Xcode**.

Ele é mais adequado como um assistente de desenvolvedor incorporado diretamente no Xcode, fornecendo conclusão de código, chat, revisão de código, Modo Agente e outros recursos. Se suas necessidades se inclinam mais para colaboração multi-agente, revisão de código, geração de testes, manutenção de documentação e fluxos de trabalho de DevOps, você também pode olhar para o **Ruflo**.

## 5. Se eu quiser incorporar um Assistente de IA na interface do meu próprio produto, qual devo escolher?

Se você está construindo um produto SaaS, aplicativo Web ou produto interno e deseja incorporar um Assistente de IA diretamente na interface do aplicativo, você pode olhar primeiro para o **CopilotKit**.

O CopilotKit é mais adequado para construir Copilotos em aplicativos. Ele permite que os Assistentes de IA leiam o estado do aplicativo, chamem ferramentas, gerem interfaces e adicionem fluxos de trabalho de confirmação humana em etapas importantes. Para equipes que desejam adicionar Copilotos de IA, UI generativa ou fluxos de trabalho de Agentes aos produtos, é uma escolha melhor.

## 6. Como devo escolher se quero construir um Assistente de IA pessoal de código aberto?

Leon se inclina mais para a direção de assistente pessoal e é adequado para tarefas pessoais, interação por voz, chamada de ferramentas, memória e cenários de automação. É adequado para usuários que valorizam privacidade, auto-hospedagem, memória de longo prazo e extensibilidade. Se você quer um assistente que possa gradualmente se conectar ao seu ambiente local e fluxos de trabalho pessoais, Leon está mais próximo dessa direção do que uma ferramenta de chat simples.

## 7. O NocoBase é apenas para empresas? Usuários individuais podem usá-lo?

**NocoBase** não é apenas para empresas. Usuários individuais também podem usá-lo para construir seus próprios sistemas de gerenciamento ou bases de conhecimento. Sua edição de código aberto é gratuita e amigável para usuários individuais.

Por exemplo, indivíduos podem usar o NocoBase para construir bases de conhecimento, registros de leitura, sistemas de gerenciamento de tarefas, bibliotecas de recursos, painéis de projetos, tabelas de acompanhamento de clientes, bibliotecas de tópicos de conteúdo e outros sistemas. Contanto que suas necessidades envolvam tabelas de dados, campos, relacionamentos, páginas e organização de longo prazo, o NocoBase pode servir como uma plataforma de gerenciamento pessoal leve.
