## Preguntas Frecuentes

## 1. ¿Qué se debe considerar al elegir un Asistente de IA de código abierto?

Al elegir un Asistente de IA de código abierto, puede centrarse en cinco aspectos: si el proyecto se mantiene activamente, si admite su caso de uso, si puede conectarse a herramientas o sistemas existentes, si admite autoalojamiento o implementación privada, y si tiene documentación clara y una comunidad activa.

Los usuarios individuales pueden priorizar la facilidad de uso, el soporte de modelos, la llamada a herramientas locales y la experiencia de escritorio. Los usuarios empresariales deben centrarse más en permisos, estructuras de datos, automatización de flujos de trabajo, integración de sistemas y capacidades de mantenimiento a largo plazo.

## 2. Si quiero construir sistemas empresariales internos para una empresa, ¿qué proyecto de Asistente de IA debo elegir?

Si su objetivo es construir sistemas empresariales internos como CRM, sistemas de aprobación, sistemas de tickets, gestión de proyectos, gestión de inventario y backends de operaciones, puede considerar **NocoBase** primero.

NocoBase es más adecuado para admitir datos comerciales empresariales, permisos, flujos de trabajo y configuración de páginas. No solo puede ayudar a construir sistemas con IA, sino que también puede continuar ajustando campos, páginas, permisos y flujos de trabajo a través de una interfaz sin código después de la generación. Para sistemas internos empresariales que requieren mantenimiento a largo plazo, este tipo de plataforma estructurada es más confiable.

## 3. Si solo necesito un Asistente de IA de escritorio para uso personal, ¿cuál debo elegir?

Si lo usa principalmente de forma personal y desea usar múltiples modelos, procesar archivos, llamar a herramientas y organizar información desde una aplicación de escritorio, puede considerar **DeepChat** o **Everywhere** primero.

DeepChat es más adecuado para reunir múltiples modelos, MCP, Habilidades y capacidades de control remoto en una sola aplicación de escritorio. Everywhere enfatiza la conciencia del contexto de la pantalla y la comprensión contextual, lo que lo hace adecuado para leer, traducir, solucionar problemas y resumir páginas web o documentos.

## 4. Si soy desarrollador y quiero usar un Asistente de IA dentro de las herramientas de desarrollo, ¿cuál debo elegir?

Si es un desarrollador en el ecosistema de Apple y usa principalmente Xcode para construir proyectos de iOS, macOS, Swift u Objective-C, puede considerar **GitHub Copilot for Xcode**.

Es más adecuado como asistente de desarrollador integrado directamente en Xcode, que proporciona finalización de código, chat, revisión de código, Modo Agente y otras capacidades. Si sus necesidades se inclinan más hacia la colaboración multiagente, la revisión de código, la generación de pruebas, el mantenimiento de documentación y los flujos de trabajo de DevOps, también puede considerar **Ruflo**.

## 5. Si quiero integrar un Asistente de IA en la interfaz de mi propio producto, ¿cuál debo elegir?

Si está construyendo un producto SaaS, una aplicación Web o un producto interno y desea integrar un Asistente de IA directamente en la interfaz de la aplicación, puede considerar **CopilotKit** primero.

CopilotKit es más adecuado para construir Copilotos dentro de aplicaciones. Permite que los Asistentes de IA lean el estado de la aplicación, llamen a herramientas, generen interfaces y agreguen flujos de trabajo de confirmación humana en pasos clave. Para equipos que desean agregar Copilotos de IA, UI generativa o flujos de trabajo de Agentes a los productos, es una mejor opción.

## 6. ¿Cómo debo elegir si quiero construir un Asistente de IA personal de código abierto?

Leon se inclina más hacia la dirección de asistente personal y es adecuado para tareas personales, interacción por voz, llamada a herramientas, memoria y escenarios de automatización. Es adecuado para usuarios que valoran la privacidad, el autoalojamiento, la memoria a largo plazo y la extensibilidad. Si desea un asistente que pueda conectarse gradualmente a su entorno local y flujos de trabajo personales, Leon está más cerca de esa dirección que una simple herramienta de chat.

**NocoBase** no es solo para empresas. Los usuarios individuales también pueden usarlo para construir sus propios sistemas de gestión o bases de conocimiento. Su edición de código abierto es gratuita y amigable para usuarios individuales.

Por ejemplo, los individuos pueden usar NocoBase para construir bases de conocimiento, registros de lectura, sistemas de gestión de tareas, bibliotecas de recursos, paneles de proyectos, tablas de seguimiento de clientes, bibliotecas de temas de contenido y otros sistemas. Siempre que sus necesidades involucren tablas de datos, campos, relaciones, páginas y organización a largo plazo, NocoBase puede servir como una plataforma de gestión personal ligera.
