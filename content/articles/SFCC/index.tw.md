中石化第五建設有限公司（以下簡稱「五建公司」）是石化行業的重要參與者，在資訊總監董可先生的領導下，正使用 NocoBase 作為其數位轉型的一部分。讓我們深入了解這個故事！

## 1. **背景與挑戰**

中石化第五建設有限公司是中國最早從事石化建設的大型施工企業之一。公司的主要業務之一是為國內外地區建造石化基礎設施，包括煉油、化工、天然氣、儲運等。五建公司實施的一個典型項目是科威特的新煉油廠項目，合約金額高達 5.19 億美元。該系統投入使用後，科威特的石油產量每年將增加 3150 萬噸[[1](http://industry.people.com.cn/n1/2019/0410/c413883-31023309.html)]。

![石化行業](https://static-docs.nocobase.com/296cb1e212172bd83f826ef2d78610c6.PNG)

作為石化建設的重要參與者，五建公司也是業內最早擁抱資訊化和數位化的企業之一。技術帶頭人董可先生親身經歷了五建公司一段重要的數位轉型時期。

傳統產業的數位轉型可以追溯到 20 世紀 90 年代中期，當時資訊和自動化技術快速發展。經過數十年的發展，行業 ERP 系統市場日趨成熟，湧現出成千上萬的解決方案。儘管如此，石化行業仍然難以找到一款通用的數位軟體或解決方案。

這主要是由於該行業的特殊性。正如董先生所說：「（問題在於）軟體供應商無法理解石化行業的具體需求，而我們又沒有專業的軟體開發人員將我們複雜的需求轉化為程式碼。」

因此，董先生和他的團隊不得不從零開始，探索各種技術，並嘗試使用 Access 和 VB 巨集語言來構建系統的 Alpha 版本。這個解決方案成本低、上手快，幫助五建公司進入了數位化的快車道。

在初期階段，這個解決方案是有效的，並且具有成本效益。隨著項目進入中期階段，Access 和 VB 巨集語言的缺點開始顯現。首先，Access 資料庫非常適合中小型資料庫應用。然而，在處理大量數據或多用戶同時訪問時，可能會出現效能問題。其次，為了滿足快速變化的業務需求而更新系統，需要進行大量的手動程式設計工作，導致維護成本增加和錯誤風險上升。

## 2. **尋找替代方案**

轉捩點發生在董先生為滿足五建公司新管理工作台項目日益增長的需求而尋找新的軟體解決方案時。該項目旨在將全面品質管理的原則應用於數位管理系統，該系統整合了影響產品品質的五個關鍵因素：人員、機器、原材料、方法和環境。

經過多年的實踐，董可先生更傾向於內部開發。由於沒有現成的解決方案，他轉向開源項目或開發框架，以盡量減少從零開始的成本和問題。

他使用了一系列工具來實現系統的不同元素：使用 Ant Design Pro 設計系統使用者介面，使用 Formily 實現表單管理，使用 G2Plot 工具呈現圖表等。然而，要完成整個系統的開發還有很長的路要走。

想像一下用現成的工具建造房子：鏟子、錘子、袋裝水泥和磚塊，但你仍然需要一磚一瓦地建造整棟房子。

軟體開發也是如此。在使用之前，需要實現許多功能，包括使用者系統、權限管理、圖表庫調用、表單庫引用等。

就在董先生在 GitHub 上繼續探索合適的開源工具時，NocoBase 以其以下亮點吸引了他的注意：

* 數據模型驅動：符合系統開發的邏輯，即數據建模優先。
* 所見即所得編輯器：使得構建表單和頁面變得非常容易。
* 即插即用：可以開發和安裝新的插件來實現新的需求。

他認為，如果他的團隊能夠基於 NocoBase 開發五建公司的系統，將有助於解決當前的痛點，從而顯著減少系統開發的工作量和複雜性。

讓我們重新審視建造房子的比喻。預製構件和自動化施工設備已經改變了建築過程。不再是一磚一瓦，而是可以使用自動化設備快速組裝預製的房屋部件，從而實現精確高效的施工。這種方法顯著減少了建築所需的人工和時間。

使用所見即所得編輯器，五建公司的一般需求也可以通過 NocoBase 的無程式碼部分，透過簡單的拖放操作快速實現。對於客製化功能，董先生的團隊可以根據專用的[開發文檔](https://docs.nocobase.com/development)開發新的插件。**這種靈活性得益於 NocoBase 的微核心和插件架構。**

## 3. 進展與成果

在過去兩年 NocoBase 發展的同時，董先生負責五建公司海外分支機構的系統創新和內部開發。在此期間，他帶領團隊完成了五建公司關鍵項目之一——**新加坡 CRISP 項目**的系統改進和推廣。

NocoBase **簡潔且標準**的 API 設計規範，促進了通過 HTTP RESTful API 快速輸入和輸出服務資源。此外，NocoBase 可以快速擴展並整合第三方服務，允許在系統內外靈活地互動和調度資源。因此，董先生和他的團隊可以高效地處理數據請求和業務邏輯，並根據需求選擇最合適的互動模式。

目前，作為新加坡 CRISP 項目一部分的人力資源工作效率管理系統，已使用 NocoBase 開發並平穩運行。這個案例產生了寶貴的最佳實踐，將顯著提升公司在全公司範圍內推廣的成功率。

### 系統技術架構

**NocoBase - 控制與應用中心** 是構建、開發和管理應用程式以及整合不同系統的核心中樞。

* 認證插件：負責認證和授權，以確保系統安全。
* 使用者與權限：管理使用者及其權限，確保不同使用者擁有適當的訪問權限。
* 資源 API：提供外部和內部的資源訪問接口，支援 RESTful API 和內部服務調用 (RPC)。
* 數據可視化：將數據可視化，用於分析和決策。
* 自動化工作流程：自動化業務流程以提高生產力。
* 應用程式（業務應用）：有效滿足業務場景的多樣化需求。

**整合的第三方平台：**

* Cloudflare Workers：處理 HTTP 請求，以提高內容分發速度和應用程式響應時間。
* Lark 整合平台：整合 Lark 服務（一款即時通訊應用），實現公司內的協作與溝通。
* Supabase：作為後端服務提供資料庫存儲和認證。
* Logto：負責認證和授權，以確保系統安全。

![技術架構](https://static-docs.nocobase.com/ed5586e6e972ff5beaa41548e55b844e.png)

董先生分享道：「NocoBase 最好的功能是一鍵切換配置模式以及全面的 [HTTP API 支援](https://docs.nocobase.com/handbook/api-doc)。」

NocoBase 允許開發者點擊右上角的模式切換按鈕，從開發模式切換到產品模式。因此，開發者可以在幾分鐘內構建一個頁面。

開發者可以通過 API 請求與數據互動，輕鬆地將 NocoBase 整合到他們現有的系統中，從而實現可自訂和擴展的數據處理能力。

例如，董先生的團隊利用 NocoBase 開發了用於監控五建公司人力資源效率的數據儀表板。該系統通過 NocoBase API 連接了大約 60 台考勤機。原始數據通過 API 收集後，使用 NocoBase 工作流程中的自動化業務流程管理進行處理，然後傳送到管理者的儀表板。

以前，數據收集、自動處理和可視化的總成本超過 100 萬元人民幣（約 14 萬美元）。**使用 NocoBase 可以將成本降低 85%。**
