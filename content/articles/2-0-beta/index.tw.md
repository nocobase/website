經過 NocoBase 團隊及所有貢獻者 8 週的努力，NocoBase 2.0-beta 已如期發布。此版本包含了對 NocoBase 2.0 新功能的廣泛打磨與測試，使 2.0 進入更穩定的階段。我們歡迎早期採用者使用此版本進行生產測試，並及時回饋任何問題或建議。

2.0-rc 將在 2026 年 2 月 11 日前發布，屆時將有更全面的測試和更多新功能。敬請期待。

## 新功能

### AI 員工

我們對 AI 員工進行了重大改進，包括：

- 優化內建 AI 員工的能力
- 支援存取外部資料來源
- 支援查詢大量資料
- 在對話內容中保留原始格式
- 改進錯誤訊息

使用手冊：[AI 員工](https://v2.docs.nocobase.com/ai-employees)

### UI 模板

2.0 beta 引入了全新的 UI 模板功能。每種模板類型都支援複製和引用，包括：

- 區塊模板
- 欄位模板
- 彈窗模板

值得注意的是，彈窗模板允許您將整個彈窗內容（子頁面中的所有頁籤和區塊）儲存為模板，大幅減少了從 1.x 版本以來的重複工作。

### 多應用 2.0

多應用管理是 NocoBase 的統一應用管理解決方案，用於在一個或多個執行環境中建立和管理多個物理隔離的 NocoBase 應用實例。透過應用管理員，使用者可以從統一的入口建立和維護多個應用程式，滿足不同業務和規模的需求。NocoBase 支援三種部署架構：

- **單一應用**：僅部署一個 NocoBase 實例，所有業務功能、資料和使用者都在同一個應用程式中執行。
- **共享記憶體多應用**：多個應用程式可以在一個 NocoBase 實例中同時執行。每個應用程式都是獨立的，可以連接到自己的資料庫，並且可以單獨建立、啟動和停止，但它們共享相同的程序和記憶體空間。使用者仍然只需要維護一個 NocoBase 實例。
- **多環境混合部署**：部署一個 NocoBase 作為統一管理中心，同時部署多個 NocoBase 實例作為應用程式執行環境來執行業務應用程式。不同的應用程式可以在不同的節點或叢集上執行。

使用手冊：[多應用](https://v2.docs.nocobase.com/multi-app/multi-app)

![](https://static-docs.nocobase.com/202512231215186.png)

### 多空間

多空間插件實現邏輯隔離，可在單一應用程式實例內建立多個獨立的資料空間，適用於多工廠或多門市運營等業務流程和系統配置高度一致的場景。

使用手冊：[多空間](https://v2.docs.nocobase.com/multi-app/multi-space)

### 資料可視化 2.0

1.x 版本提供了基於 Ant Design Charts 和 ECharts 的基本資料可視化插件，用於基礎的資料可視化。然而，在處理複雜圖表、個人化互動和資料聯動場景方面有所不足。

NocoBase 2.0 引入了基於 ECharts 的全新資料可視化插件，支援可視化配置模式以及 SQL 和 JSON 模式。它支援事件，可輕鬆處理複雜圖表、多圖表聯動和資料下鑽場景。

不用擔心 SQL 和 JSON 太複雜——AI 員工 Nathan 會提供協助。

使用手冊：[資料可視化](https://v2.docs.nocobase.com/data-visualization)

![](https://static-docs.nocobase.com/20251102221207-2025-11-02-22-12-09.png)

### 工作流程

#### 新增「多條件分支」節點

類似於程式語言中的 `switch / case` 或 `if / else if` 語句。系統會依序評估多個配置的條件，並僅執行滿足條件的分支。此節點將取代「並行分支」節點的部分功能。我們建議使用新的「多條件分支」節點來取代同時使用條件和並行節點的工作流程。

![20251123224209](https://static-docs.nocobase.com/20251123224209.png)

使用手冊：[多條件分支](https://v2.docs.nocobase.com/workflow/nodes/multi-conditions)

### 審批 2.0

審批插件中的大部分區塊配置已使用 FlowEngine 進行重構，支援更靈活的配置和使用。同時也新增了多項功能。

使用手冊：[審批](https://v2.docs.nocobase.com/workflow/triggers/approval)

#### 在審批處理介面中選擇顯示資料快照或最新狀態

* **快照**
  在審批過程中，申請人和審批人看到的是他們進入時的記錄狀態，提交後只能看到自己修改的記錄——之後其他人所做的更新將不可見。
* **最新**
  在審批過程中，無論記錄在其操作前的狀態如何，申請人和審批人在整個過程中始終看到記錄的最新版本。流程結束後，他們將看到記錄的最終版本。

#### 支援配置發起人資料範圍

您可以配置基於使用者範圍的權限，以決定哪些使用者可以發起審批。

![20251226114623](https://static-docs.nocobase.com/20251226114623.png)

「我發起的」卡片允許靈活配置審批資訊和業務資料欄位。

- **多來源統一接入**：公開表單、客戶入口網站、郵件解析、API/Webhook
