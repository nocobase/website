## 功能比較

在功能方面，**NocoBase** 和 **Refine** 各自擁有獨特的優勢，以滿足不同使用者的需求。我們將從四個關鍵領域進行比較：CRUD 功能、資料模型與存取控制、前端開發與自訂元件，以及工作流程自動化。

### CRUD

**NocoBase** 和 **Refine** 都是優秀的 CRUD（建立、讀取、更新、刪除）工具，專為管理資料、處理資料庫操作以及建構商業應用程式而設計。

💡 深入探討：[GitHub 星數最高的 8 大開源 CRUD 專案](https://www.nocobase.com/en/blog/crud-projects)

**NocoBase** 提供了一套完整的 CRUD 功能，讓使用者無需編寫任何程式碼即可輕鬆建立、讀取、更新和刪除資料。

![CRUD.PNG](https://static-docs.nocobase.com/5dbbe8b0fd9426209ad871496b7c4ea3.PNG)

以下是 **NocoBase** 在 CRUD 操作中的一些核心優勢：

1.  **直觀的圖形化介面**：NocoBase 提供可視化介面，讓使用者能透過拖放操作來設計和管理資料庫表格。使用者可以輕鬆配置表格結構、欄位類型和關聯，同時即時管理資料，大幅簡化了資料庫管理流程。
2.  **模組化資料管理**：NocoBase 支援模組化的 CRUD 操作，讓使用者能透過表格、資料來源和欄位模組來建立複雜的商業資料模型。其靈活的資料模型設計允許使用者自訂表單、列表視圖等，適用於多樣化的資料管理需求。
3.  **基於角色的存取控制**：在執行 CRUD 操作時，NocoBase 允許基於角色的權限管理，確保不同使用者對資料擁有適當的存取和控制權限。此功能在企業應用程式中尤為重要，有助於保護資料安全並維持合規性。
4.  **動態資料顯示與操作**：NocoBase 為 CRUD 互動提供了動態介面，使使用者能透過表單或視圖快速管理資料。它還支援批次處理和管理大規模資料，非常適合希望快速建構內部業務管理系統的團隊。

🙌 立即體驗：[NocoBase 教學 – 任務管理系統](https://www.nocobase.com/en/tutorials/task-tutorial-introduction)

同樣地，**Refine** 在 CRUD 操作方面也極具能力。

![CRUD.PNG](https://static-docs.nocobase.com/4d415ba36f35b58563b483f10b15daee.PNG)

**Refine** 在 CRUD 開發中的關鍵特性：

1.  **React 帶來的高度靈活性**：Refine 讓開發者能以最少的程式碼實現 CRUD 功能。透過使用 `useData` hooks 和內建的資料管理功能，開發者可以輕鬆整合 REST API、GraphQL 或其他資料來源，以動態獲取和操作資料。
2.  **自訂表單與資料顯示**：開發者可以利用 Refine 的 UI 元件（例如 Ant Design 或 Material UI）快速建立和管理資料輸入表單及顯示頁面。這種高度可自訂的前端操作方式，允許開發者根據業務需求調整 CRUD 介面的佈局和功能。
3.  **狀態管理與資料互動**：Refine 透過 React 的狀態管理機制提供了流暢的資料處理體驗。開發者可以輕鬆管理 CRUD 操作，並將其與使用者互動無縫連接。這種低程式碼方法減輕了開發者的工作量，同時提供了對前端功能的強大控制。
4.  **模組化設計**：Refine 的模組化結構允許開發者為不同的頁面或元件建立獨立的 CRUD 功能，這適用於需要多個模組化資料管理頁面的專案。開發者可以重複使用元件和邏輯，快速開發複雜應用程式的不同部分。

### 資料模型與存取控制

資料模型是 CRUD 操作的骨幹，定義了資訊在系統中如何被結構化、儲存和關聯。高效的資料模型管理確保使用者能無縫地建立、讀取、更新和刪除資料，使應用程式能如預期般運作。同樣重要的是存取控制，它決定了誰可以存取和修改特定資料。對於任何企業級應用程式而言，確保不同使用者群組擁有適當的存取權限，對於維護資料安全性和系統完整性至關重要。

**NocoBase** 的核心優勢之一在於其**靈活的資料模型**和強大的存取控制系統。

![flexible data model.png](https://static-docs.nocobase.com/533f86a309c52b11a57d2d59376f1987.png)

**NocoBase** 基於關聯式資料庫設計，允許使用者建立複雜的資料結構，並透過其圖形化介面靈活配置表格之間的關聯。對於開發者而言，NocoBase 提供了可自訂的資料管理選項，讓使用者能根據業務需求擴展和調整資料模型，而無需編寫複雜的程式碼。

![customizable data management.png](https://static-docs.nocobase.com/8ba8fefe0f52ffd8e088ac0b95ea42cb.png)

此外，**NocoBase** 提供了全面的存取控制和使用者驗證功能。使用者可以根據角色和特定操作，為資料設定精細的權限，確保不同使用者群組之間的存取權限嚴格分離。

![access control.png](https://static-docs.nocobase.com/143cad1bf26ff415206a5374d7cf55a4.png)

這個強大的存取控制系統對於企業應用程式尤其關鍵，能確保資料安全與法規遵循。NocoBase 透過插件提供多種使用者驗證方式，包括簡訊驗證、SAML SSO、OIDC SSO 和 CAS SSO。使用者可以利用 **Google Workspace** 或 **Microsoft Azure** 等平台進行身分登入系統，或整合 **Auth0**、**Logto** 和 **Keycloak** 等工具。此外，開發者可以透過 NocoBase 的基礎介面輕鬆擴展驗證選項，以滿足其特定需求。

相比之下，**Refine** 主要透過後端服務整合來處理資料模型，結合了資料檢索和存取控制的靈活性。

![backend service integration.png](https://static-docs.nocobase.com/81befa3d5f00825d25d02c1f15089565.png)

憑藉 Refine 的資料檢索能力，開發者可以輕鬆地與各種後端 API（如 **REST API**、**GraphQL**）互動，或整合 **Strapi**、**Supabase** 和 **Airtable** 等服務。

![backend APIs.png](https://static-docs.nocobase.com/20b3a853d2d5b725648f48d3c2f29a4d.png)

Refine 提供了一個無縫的資料檢索模型，能自動管理請求狀態（例如載入中、錯誤、成功），讓開發者能專注於業務邏輯，而非資料獲取的複雜細節。

![seamless data retrieval model.png](https://static-docs.nocobase.com/a7a5db98116ef9da6434f3bc3b951f72.png)

在存取控制方面，Refine 提供了一個靈活的驗證和授權機制。Refine 支援多種常見的驗證方法，例如 **OAuth**、**JWT** 或自訂驗證方案。這使得開發者可以透過與後端服務整合，並根據專案需求配置不同的驗證流程，輕鬆管理使用者驗證。

### 前端開發與自訂元件

在前端開發與自訂元件方面，**NocoBase** 和 **Refine** 有著不同的關注點。

![WYSIWYG.png](https://static-docs.nocobase.com/c1caccbc7c0270c10e943867e102223d.png)

**NocoBase** 提供了一個直觀的**所見即所得 (WYSIWYG)** 使用者介面編輯器，讓使用者能透過簡單的拖放操作快速建構和配置前端介面，無需編寫程式碼。這個編輯器為使用者提供了自訂頁面佈局、區塊顯示和資料視圖的靈活性，極大地簡化了前端開發任務——特別是對於沒有前端開發經驗的使用者。

**NocoBase UI 編輯器**的主要優勢包括：

![UI editor.png](https://static-docs.nocobase.com/13174618869b9eb4ab69b379961cb3b9.png)

1.  **模組化設計**：使用者可以使用預先定義的區塊（如表單、表格、卡片和列表）快速建構應用程式介面。這些元件可以透過拖放靈活組合，幫助使用者根據特定需求自訂頁面佈局。
2.  **即時預覽與快速除錯**：NocoBase 提供即時預覽功能，讓使用者在設計過程中就能查看介面效果，並根據需要進行調整。這種快速回饋機制顯著提高了開發效率，特別是在快節奏、迭代頻繁的業務環境中。
3.  **自訂性與可擴展性**：雖然 NocoBase 的介面編輯器非常適合無程式碼使用者，但它也為有程式碼經驗的使用者提供了透過插件進行自訂整合和開發的支援。

![visual interface design.gif](https://static-docs.nocobase.com/318c29d41b5be6852d8481e6e8822304.gif)

這種高度視覺化的介面設計方法，幫助使用者快速實現頁面佈局和功能，同時也支援複雜的互動邏輯和動態資料顯示。

相比之下，**Refine** 更適合具有前端開發經驗的開發者，讓他們能以最少的程式碼建構前端介面。**Refine** 提供了一個高度模組化的元件系統，使開發者能夠使用流行的 UI 庫，例如 **Ant Design** 和 **Material UI**。

![modular component system.png](https://static-docs.nocobase.com/7369b60625ce93759dddea0eb4f613e1.png)

Refine 的元件化設計，以及與 **React** 生態系統的無縫整合，為開發者在建構頁面時提供了極大的靈活性。開發者可以自訂元件、定義複雜的頁面邏輯，並高效地重複使用和組合元件。此外，Refine 的佈局和狀態管理系統簡化了開發流程，讓開發者能專注於業務邏輯，而無需擔心底層的狀態管理細節。

![Reactecosystem.png](https://static-docs.nocobase.com/5aaf8c3d4b53bc9000990ec2238b9b93.png)

在複雜的前端開發場景中，**Refine** 提供了更大的自由度和靈活性，而 **NocoBase** 則專注於提供圖形化介面和預先定義的元件，幫助使用者更高效地建構前端介面。

### 工作流程與自動化

在工作流程與自動化方面，**NocoBase** 提供了強大的業務流程配置與自動化能力。與需要自訂程式碼來實現複雜業務邏輯的低程式碼平台不同，NocoBase 的圖形化工作流程編輯器讓使用者只需點擊幾下，就能設計、配置和自動化整個業務流程。

![automation.png](https://static-docs.nocobase.com/f56a67f876450a3343a6db6b0b9ec3c9.png)

**NocoBase 的工作流程功能**基於可視化的流程設計器，讓使用者能夠透過定義觸發器、動作和工作流程節點來實現複雜的業務自動化。其核心優勢包括：

1.  **圖形化工作流程設計**：NocoBase 提供直觀的圖形化介面，每個步驟的執行邏輯都清晰可見。無論是審批流程、自動任務分配，還是跨系統資料處理，使用者都可以透過簡單的設定來配置這些流程，實現進階的工作流程控制。
   ![Graphical Workflow Design.png](https://static-docs.nocobase.com/77f34ec214a8d37a0e4a54557936e770.png)
2.  **觸發器與條件分支**：NocoBase 支援多種事件觸發器，允許工作流程根據資料變更、時間條件或手動操作來啟動。結合條件分支，使用者可以為每個節點設定不同的條件，從而根據業務邏輯動態選擇下一步操作。
3.  **自動化任務**：使用者可以自動化一系列任務。例如，當新增或更新資料時，系統可以自動發送通知、更新相關記錄，或呼叫外部 API 以觸發其他系統中的操作。
4.  **外部整合與 API 呼叫**：NocoBase 的工作流程不僅限於內部操作，還可以與外部系統整合。使用者可以在工作流程中配置外部 API 呼叫，使平台能夠自動與第三方服務互動。這項能力使 NocoBase 在處理跨系統業務場景時具有顯著優勢。

相比之下，**Refine** 作為一個前端開發框架，並未直接提供內建的工作流程管理功能。然而，Refine 擅長**與外部工作流程引擎或後端自動化工具無縫整合**。開發者可以使用 Refine 建構自訂的前端介面，並將其與 **n8n**、**Integromat** 等後端工作流程系統整合，以完成複雜的自動化任務。

## 整合能力與可擴展性

在整合與可擴展性方面，**NocoBase** 和 **Refine** 提供了不同的方法來滿足使用者需求。**NocoBase** 可以快速擴展前端功能以及與第三方服務的後端整合，涵蓋從前端到後端的完整業務邏輯。另一方面，**Refine** 主要專注於前端可擴展性，允許與外部 API 和資料來源整合，但缺乏直接的後端可擴展性，需依賴外部服務（例如 Strapi、Supabase）來處理資料和業務邏輯。

### NocoBase：全面的整合能力

**NocoBase** 的插件架構提供了強大的可擴展性，使用者和開發者只需安裝並啟用插件，即可快速整合所需的功能和第三方服務。NocoBase 的所有核心功能（如資料管理、存取控制和工作流程）都是透過插件提供的。這種模組化設計不僅簡化了系統維護，也允許使用者根據特定需求自由選擇和配置功能。

![Integration Capabilities.png](https://static-docs.nocobase.com/97782eb52aee16623ab595e159ee95a1.png)

*   **插件開發與可擴展性**：NocoBase 支援開發自訂插件，使開發者能夠根據業務需求編寫插件來擴展系統功能。例如，他們可以整合新的資料來源、外部 API 或自訂業務邏輯。這種靈活的方法最大限度地減少了修改核心系統程式碼的需求，降低了風險並維護了系統完整性。
*   **第三方服務整合**：NocoBase 透過插件實現與第三方服務（如外部資料庫和驗證服務）的無縫整合。使用者可以直接呼叫這些服務並將其嵌入到業務流程中。例如，整合 **OAuth** 進行使用者驗證，或連接到外部使用者管理系統，都可以透過插件實現，大大降低了實施的複雜性，無需開發經驗。

💡 深入探討：[深入探討：低程式碼/無程式碼整合能力](https://www.nocobase.com/en/blog/low-code-no-code-integration)

這種基於插件的架構增強了 NocoBase 的可擴展性，使其能夠隨著業務需求的發展擴展功能，而無需對底層程式碼進行大規模更改，確保了穩定性和靈活性。

### Refine：專注於前端開發與客製化

**Refine** 在前端開發和客製化方面表現出色，它利用其 **React** 框架在建構前端功能時提供了高度的靈活性。Refine 允許開發者利用 React 生態系統和現有的元件庫來實現更客製化的前端功能，如下方即時協作像素藝術平台的範例所示。

![Frontend Development and Customization.png](https://static-docs.nocobase.com/4cdf0deee9a59640d4dfd6c09baca397.png)

*   **自訂前端元件**：Refine 允許開發者根據專案需求自訂和重複使用 React 元件，從而能夠建立高度複雜的前端應用程式。透過整合 **Ant Design** 或 **Material UI** 等 UI 庫，開發者可以快速建立現代化的介面，並靈活調整以滿足業務需求。例如，他們可以根據使用者角色自訂頁面佈局、動態渲染不同的資料顯示，或使用自訂表單元件擴展互動功能。
*   **模組化開發**：Refine 提供了一種模組化的開發方法，允許開發者為專案的不同部分建立獨立的模組，並根據需要擴展功能。這使得在大型專案中的協作更加高效，團隊可以同時處理不同的模組。此外，Refine 的程式碼結構清晰且遵循開發標準，使得未來的維護和迭代更加直接。
*   **外部函式庫與工具的無縫整合**：由於 Refine 完全基於 React，開發者可以輕鬆整合 React 生態系統中的任何外部函式庫和工具。無論是資料視覺化工具、狀態管理庫，還是複雜的路由和存取控制邏輯，開發者都可以根據專案需求靈活選擇並將其整合到 Refine 專案中。

### 範例：如何整合外部 API 或服務

在 **NocoBase** 中，整合外部 API 和服務是透過其插件機制靈活實現的。開發者可以建立自訂插件，將外部 API 資料與 NocoBase 的資料模型、介面或業務流程整合，獨立於工作流程操作。

![Integrate External APIs or Services.png](https://static-docs.nocobase.com/64386b8d8a3df3d4deb511c54f149ec6.png)

典型的流程包括：

1.  **開發自訂 API 插件**：開發者建立自訂插件，根據業務需求呼叫外部 API 並處理返回的資料。
2.  **與資料模型或介面整合**：該插件可以將外部 API 資料直接整合到 NocoBase 的資料模型或使用者介面中進行操作和顯示，無需依賴工作流程系統。
3.  **與工作流程結合（可選）**：對於自動化場景，開發者可以結合工作流程功能，使用觸發器（例如資料變更或使用者操作）自動呼叫外部 API，進一步提高效率。

這種基於插件的架構提供了靈活性，允許使用者根據不同的業務需求決定是否結合工作流程。外部 API 資料可以直接用於介面顯示、資料處理或業務流程管理，滿足多樣化的整合需求。

在 **Refine** 中，開發者可以直接使用 React 元件呼叫外部 API 來檢索和操作資料。透過使用自訂 hooks 或內建的 `useData` 功能，開發者可以輕鬆獲取外部 API 資料並進行處理。例如，在建構 CRM 應用程式時，開發者可以：

1.  使用 Refine 的資料檢索功能從 **REST API** 或 **GraphQL** 獲取客戶資料。
2.  使用自訂的表格或圖表元件顯示資料。
3.  使用 React 狀態管理工具管理資料操作（CRUD），並將更新同步到外部服務。

兩個平台都提供了穩固的可擴展性，但各有側重——NocoBase 透過其插件架構提供全面的後端和前端整合，而 Refine 則透過其基於 React 的模組化設計強調前端客製化和靈活性。

## 使用案例總結

從以上分析可以看出，**NocoBase** 和 **Refine** 各有側重和優勢，為不同的使用者需求提供了量身定制的解決方案。無論是開發方法還是可擴展性，這兩款工具都為不同的使用者提供了合適的解決方案。

### NocoBase 適用於以下場景：

1.  **快速開發內部業務系統**：NocoBase 非常適合快速開發和迭代 CRM、RMS（記錄管理系統）或 MES（製造執行系統）等業務系統。這些系統可以透過簡單的配置或插件快速建構，允許非技術使用者參與開發，顯著縮短開發週期並降低成本。
2.  **企業數位轉型**：對於希望實現數位轉型、提高效率並降低成本的企業，NocoBase 提供了一個靈活且可擴展的解決方案。其插件擴展能力使其能夠與外部系統無縫整合，讓企業能夠快速回應新需求。
3.  **技術與非技術使用者的協作**：NocoBase 支援技術團隊透過插件開發進行深度客製化，同時也允許非技術使用者使用無程式碼工具建構基本功能。這種靈活性使其成為業務團隊與技術團隊協作的理想平台。

### Refine 更適用於以下場景：

1.  **複雜前端介面的客製化開發**：對於需要高度客製化前端介面和互動邏輯，甚至是遊戲風格介面的專案，Refine 提供了強大的靈活性。它允許開發者利用 React 生態系統和元件庫進行高度客製化，適用於建構複雜的多頁面應用程式。
2.  **開發者主導的專案**：Refine 更適合擁有前端開發專業知識的團隊。開發者可以使用自訂程式碼和資料整合來建構複雜的業務邏輯，非常適合開發企業級的複雜應用程式。
3.  **長期、大型專案**：對於需要長期維護和頻繁迭代的專案，Refine 提供了出色的可擴展性和可維護性。它非常適合希望對程式碼庫保持嚴格控制的技術團隊。
