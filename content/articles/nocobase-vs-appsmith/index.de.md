## Datenverwaltungsfähigkeiten

Datenmanagement ist eine grundlegende Funktion in der Anwendungsentwicklung, insbesondere für Anwendungen, die komplexe Datenstrukturen und -beziehungen verarbeiten müssen. Die Stärke der Datenverwaltungsfähigkeiten einer Plattform wirkt sich direkt auf die Entwicklungseffizienz und die Wartbarkeit des Projekts aus.

![NocoBase Datenverwaltungsfähigkeiten.png](https://static-docs.nocobase.com/4f7a532c116f9aabfb516f825736a3d3.png)

**NocoBase** ist eine datenmodellgesteuerte Plattform mit leistungsstarken Datenverwaltungsfunktionen, darunter:

* **Datenmodellgetriebenes Design**: NocoBase verwendet einen datenmodellgetriebenen Ansatz, bei dem Benutzer die grundlegende Struktur der Anwendung durch die Konfiguration von Datenmodellen definieren können. Die Datenquellenverwaltung ist ein separates Plugin, das die Verwaltung verschiedener Datenquellen und Tabellen ermöglicht. Unterstützte Datenquellen umfassen die Hauptdatenbank von NocoBase (mit Unterstützung für MySQL, PostgreSQL, SQLite), externe MySQL-, MariaDB- und PostgreSQL-Datenbanken.
* **Komplexe Datenbeziehungen**: NocoBase unterstützt die Modellierung und Verwaltung komplexer Datenbeziehungen, wie Eins-zu-Eins, Eins-zu-Viele und Viele-zu-Viele. Entwickler können Beziehungen zwischen Tabellen herstellen und über die Schnittstelle oder API einfach auf verwandte Daten zugreifen oder diese manipulieren.
* **Feingranulare Zugriffskontrolle**: NocoBase bietet eine granulare Berechtigungssteuerung, die Datenzugriffseinstellungen auf Feld- und Zeilenebene ermöglicht. Entwickler können Zugriffsberechtigungen für verschiedene Rollen und Benutzergruppen konfigurieren und so Datensicherheit und Compliance gewährleisten.

![Appsmith Datenverwaltungsfähigkeiten.png](https://static-docs.nocobase.com/f53b1bb4fe7d562ce0c442c15e122ba9.png)

**Appsmith** stützt sich hauptsächlich auf seine Datenintegrationsfähigkeiten und unterstützt Verbindungen zu verschiedenen Datenquellen, wie Datenbanken (MySQL, PostgreSQL, MongoDB), REST-APIs und GraphQL.

* **Mehrere Datenquellenintegration**: Appsmith ermöglicht es Entwicklern, gängige Datenquellen schnell zu integrieren, um Daten abzufragen und anzuzeigen. Integrierte Konnektoren erleichtern die Verbindung zu Datenbanken oder Drittanbieter-APIs und ermöglichen Dateninteraktionen mit minimaler Konfiguration. Dieser Ansatz eignet sich für Szenarien, in denen keine komplexe Datenmodellierung erforderlich ist.
* **Visuelle Konfiguration für Datenoperationen**: Appsmith bietet eine intuitive visuelle Oberfläche zum Konfigurieren von Datenabfragen, Filtern, Paginierung und mehr. Dieser Ansatz vereinfacht zwar grundlegende Datenoperationen, kann aber bei der Handhabung komplexer Geschäftslogik an Flexibilität mangeln.
* **Vereinfachte Zugriffskontrolle**: Die Zugriffskontrolle von Appsmith erfolgt hauptsächlich auf Seitenebene und bietet ein grundlegendes Datenzugriffsmanagement. Für komplexe Berechtigungsstrukturen oder eine zeilenbasierte Steuerung sind möglicherweise benutzerdefinierte Logik oder externe Dienste erforderlich.

Zusammenfassend ist **NocoBase** besser geeignet für komplexe Datenmodellierung und feingranulare Zugriffskontrolle, während **Appsmith** bei der schnellen Integration mehrerer Datenquellen und der einfachen Datenanzeige glänzt.

## UI/UX-Designflexibilität

Der Aufbau einer intuitiven und effizienten Benutzeroberfläche ist entscheidend für die Verbesserung der Benutzererfahrung bei der Anwendungsentwicklung.

**NocoBase** verwendet einen WYSIWYG-Oberflächenkonfigurationsmodus. Klicken Sie auf die Schaltfläche „Oberflächenkonfiguration“ (orangefarbener Bereich) in der oberen rechten Ecke, um zwischen dem Konfigurationsmodus und dem Nutzungsmodus zu wechseln.

![WYSIWYG-Oberfläche.png](https://static-docs.nocobase.com/677b4ba98ccf4758df3f969a4672d1ba.png)

**Frontend-Blöcke**: NocoBase bietet Notion-ähnliche Frontend-Blöcke zum Erstellen modularer Elemente. Entwickler können Blöcke (wie Tabellen, Formulare, Diagramme, Kanban-Boards usw.) verwenden, um das Layout zu gestalten. Jeder Block kann mit spezifischen Datenquellenbindungen, Sichtbarkeitsbedingungen und Benutzerinteraktionen konfiguriert werden.

![Frontend-Blöcke.gif](https://static-docs.nocobase.com/a0fc873d996723fe4c50e34471954230.gif)

**Dynamisches Layout und Editor**: Die Plattform unterstützt Drag-and-Drop-Operationen zum Neuanordnen von Blöcken, Definieren der Sichtbarkeit, Binden von Datenquellen und Konfigurieren von Aktionen. Dieser dynamische Layout-Ansatz ermöglicht die Erstellung von Oberflächen, die für verschiedene komplexe Szenarien geeignet sind, wie z. B. Admin-Panels, Dashboards und Geschäftsformulare.

![Umfangreiche integrierte Blöcke und Erweiterbarkeit.png](https://static-docs.nocobase.com/8c869dc1643cb35fac7c1850e73097f8.png)

**Umfangreiche integrierte Blöcke und Erweiterbarkeit**: Der Editor von NocoBase unterstützt die Erweiterung von Funktionen durch benutzerdefinierten Code oder Plugins und bietet ein Maß an Freiheit, das nahe an der nativen Frontend-Entwicklung liegt.

**Appsmith** zeichnet sich durch Drag-and-Drop-Oberflächendesign und eine umfangreiche Bibliothek integrierter Komponenten aus.

![Appsmith-Funktionen.png](https://static-docs.nocobase.com/cb2d6fd23c1131c25597c6fa95eaad68.png)

**Drag-and-Drop-Design**: Ermöglicht die schnelle Erstellung von Benutzeroberflächen durch Ziehen von Komponenten wie Tabellen, Schaltflächen, Formularen und Diagrammen auf die Design-Leinwand.

![Umfassende Komponentenbibliothek.png](https://static-docs.nocobase.com/c845291348959c8e1afb4c4534af3806.png)

**Umfassende Komponentenbibliothek**: Die Komponentenbibliothek deckt gängige UI-Elemente ab und eignet sich für die meisten Datenanzeige- und Interaktionsanforderungen. Entwickler können diese Komponenten verwenden, ohne von Grund auf neu entwerfen zu müssen.

![Umfassende Komponentenbibliothek.png](https://static-docs.nocobase.com/3b7e6342bd602f7fa3acdbc62e9cbc44.png)

**Grundlegende Stilanpassung**: Appsmith bietet grundlegende Styling-Optionen, aber seine Anpassbarkeit ist für hochgradig personalisierte Designs eingeschränkt.

Zusammenfassend bietet **NocoBase** eine größere Designflexibilität und eignet sich für komplexe Layouts und hochgradig individualisierte Unternehmensanwendungen. **Appsmith** vereinfacht die Erstellung von Schnittstellen mit Drag-and-Design und ist ideal für schnelles Prototyping und standardisierte Schnittstellen.

## **Integrationsfähigkeiten**

Integrationsfähigkeiten wirken sich direkt auf die Skalierbarkeit und Flexibilität der Datenverarbeitung innerhalb einer Anwendung aus. Gute Integrationsoptionen ermöglichen es Unternehmen, schnell eine Verbindung zu verschiedenen Datenquellen, Unternehmensanwendungen und Cloud-Diensten herzustellen, ohne bestehende Systeme zu ändern, und gewährleisten so nahtlose Geschäftsprozesse.

💡 Tiefer eintauchen: [Low Code/No Code Integrationsfähigkeiten](https://www.nocobase.com/en/blog/low-code-no-code-integration)

**NocoBase** bietet ein flexibles Plugin-System, das es Entwicklern ermöglicht, benutzerdefinierte Integrationen und Funktionserweiterungen durch Plugins zu erreichen.

![plugins.png](https://static-docs.nocobase.com/b1791e642b55d877844426401a223a27.png)

NocoBase hat mehrere Möglichkeiten, eine Integration mit Drittanbieter-Systemen zu erreichen:

* **Drittanbieter-Datenbanken als Datenquellen**: NocoBase unterstützt die Integration externer Datenbanken (wie MySQL, PostgreSQL) als Datenquellen, sodass Entwickler direkt innerhalb der Plattform auf externe Daten zugreifen und diese manipulieren können.
* **Drittanbieter-API-Integration**: Entwickler können APIs anderer Systeme als Datenquellen integrieren, Schnittstellen konfigurieren, um Daten abzurufen oder zu pushen, und so nahtlose Dateninteraktionen mit bestehenden Systemen ermöglichen.
* **Single Sign-On (SSO):** NocoBase unterstützt SSO und ermöglicht die Integration mit Drittanbieter-Authentifizierungssystemen, sodass Benutzer über ein einheitliches Authentifizierungssystem auf mehrere Anwendungen zugreifen können.
* **Eingebettete Integration**: NocoBase unterstützt die Einbettung von Drittanbieter-Systemen in seine Oberfläche und ermöglicht so eine engere Integration zwischen Systemen.
* **Multi-Anwendungs-Integration**: Verschiedene NocoBase-Anwendungen können miteinander interagieren, was den Datenaustausch und die Zusammenarbeit zwischen Anwendungen ermöglicht und sich für den Aufbau verteilter Anwendungssysteme eignet.

**Appsmith** bietet verschiedene Integrationsoptionen und integrierte Datenquellenkonnektoren, die schnelle Verbindungen zu gängigen Datenquellen und Drittanbieter-Diensten unterstützen. Einige erweiterte Integrationsfunktionen erfordern eine Enterprise-Edition, die zusätzliche Kosten verursacht.

![enterprise.png](https://static-docs.nocobase.com/f632e95167690715b7fe0c4759d702e7.png)

* **Breite Datenquellenunterstützung**: Appsmith unterstützt gängige Datenbanken (MySQL, PostgreSQL, MongoDB usw.), REST-APIs, GraphQL, Airtable und Google Sheets über integrierte Konnektoren, sodass Entwickler externe Datenquellen mit minimaler Konfiguration einfach integrieren können.
* **Automatisierte API-Integration**: Die Plattform bietet automatisch generierte API-Integrationsvorlagen für die schnelle Einrichtung mit REST- und GraphQL-Diensten.
* **Erweiterte Integration (Enterprise Edition):** Funktionen wie private Embeds, Integration in selbst gehostete Umgebungen und erweiterte Sicherheitskontrollen sind in der Enterprise-Edition verfügbar und ideal für Projekte, die höhere Sicherheit oder komplexe Integration erfordern.
* **Benutzerdefinierte Authentifizierung und SSO (Enterprise Edition)**: Die Enterprise-Edition enthält auch benutzerdefinierte Authentifizierungsmethoden und SSO, was die Integration mit bestehenden Unternehmens-Identitätsverwaltungssystemen erleichtert.

Aus der obigen Integrationsanalyse geht hervor, dass **NocoBase** besser geeignet ist für Projekte, die benutzerdefinierte Integrationen, komplexe Datenverarbeitung und mehrstufige Geschäftsprozesse erfordern. **Appsmith** ist ideal für Projekte, die verschiedene Datenquellen oder APIs integrieren und schnell datengesteuerte Schnittstellen wie Dashboards und Berichtstools implementieren müssen.

## Zugriffskontrolle

Zugriffskontrolle ist für Unternehmensanwendungen unerlässlich, um Datensicherheit und Compliance zu gewährleisten.

**NocoBase** bietet eine feingranulare, mehrstufige Zugriffskontrolle für komplexe Sicherheitsanforderungen:

![Mehrstufiges Berechtigungsmanagement.png](https://static-docs.nocobase.com/1d91d7c99f039664b2f9646481a60c53.png)

**Mehrstufiges Berechtigungsmanagement**: Unterstützt Berechtigungen für Tabellen, Felder und Datensätze. Entwickler können verschiedene Datenzugriffsberechtigungen für verschiedene Rollen und Benutzergruppen konfigurieren und so eine präzise Datenverwaltung gewährleisten. Beispielsweise können einige Benutzergruppen nur bestimmte Felder oder Datensätze anzeigen, um unbefugten Zugriff zu verhindern.

![Rollen- und Benutzergruppenverwaltung.png](https://static-docs.nocobase.com/e2da6171ae41cdbf1f98a163b1ac9653.png)

**Rollen- und Benutzergruppenverwaltung**: Ermöglicht Entwicklern die Erstellung benutzerdefinierter Rollen und Benutzergruppen mit spezifischen Berechtigungen, was die Konfiguration flexibel und an sich ändernde Geschäftsanforderungen anpassbar macht.

![Datenzugriffsprüfung und -protokollierung.png](https://static-docs.nocobase.com/eeee0bd97823a30e7f9c5e0777c46a34.png)

**Datenzugriffsprüfung und -protokollierung**: NocoBase protokolliert auch Benutzeraktionen und ermöglicht so die Nachverfolgung für Sicherheitsaudits und Compliance-Prüfungen.

**Appsmith** bietet grundlegende Zugriffskontrollfunktionen, die sich hauptsächlich auf den Zugriff auf Anwendungs- und Seitenebene konzentrieren:

![Appsmith ACL.png](https://static-docs.nocobase.com/0b3fd41a5f99a9081e20b201ff800c56.png)

**Berechtigungseinstellungen auf Seitenebene**: Ermöglicht Entwicklern, Zugriffsberechtigungen für die gesamte Anwendung oder bestimmte Seiten festzulegen, was den Zugriff basierend auf Benutzerrollen einschränken kann. Obwohl es für grundlegende Zugriffskontrollanforderungen geeignet ist, fehlt ihm die detaillierte Datenebenenkontrolle von NocoBase.

![Rollenverwaltung.png](https://static-docs.nocobase.com/99b5f01de188c4ac0e74646d741a7f79.png)

**Rollenverwaltung**: Eine konfigurierbare rollenbasierte Zugriffskontrolle ist verfügbar, hauptsächlich jedoch für übergeordnete Strukturen und nicht für eine feingranulare Datenkontrolle.

**Erweiterungen der Enterprise Edition**: Erweiterte Funktionen wie SSO und dynamische Berechtigungen basierend auf Benutzerattributen erfordern die Enterprise-Edition, was für Unternehmen mit strengen Zugriffskontrollanforderungen zusätzliche Kosten verursachen kann.

## Workflow-Automation

Workflow-Automation ist entscheidend für die Steigerung der Geschäftseffizienz, **insbesondere bei der Bewältigung komplexer Geschäftsprozesse**, da sie manuelle Aufgaben und Fehler erheblich reduziert.

💡 Weiterlesen: [5 beste Tools zum Erstellen von Workflow-Automation](https://www.nocobase.com/en/blog/workflow-automation-tools)

**NocoBase** bietet robuste [Workflows](https://docs.nocobase.com/handbook/workflow)-Unterstützung für die Automatisierung komplexer Geschäftsprozesse:

![Visueller Workflow-Designer.png](https://static-docs.nocobase.com/1d3e5932a0ebcdd88551b7387f554a05.png)

**Visueller Workflow-Designer**: Wird mit einem Drag-and-Drop-Workflow-Editor geliefert, der es Entwicklern ermöglicht, Workflow-Schritte zu konfigurieren, einschließlich bedingter Logik, Datenmanipulation und Benachrichtigungen.

![Verschiedene Knotentypen.png](https://static-docs.nocobase.com/674c45bdd5a80da57902f1dc2568f9dc.png)

**Verschiedene Knotentypen**: Unterstützt mehrere Knotentypen, wie Bedingungsknoten, API-Aufrufknoten und Datenbankoperationsknoten. Entwickler können den geeigneten Knotentyp basierend auf den Geschäftsanforderungen auswählen und so flexible Konfigurationen von Geschäftsprozessen ermöglichen.

![Ereignisgesteuerte Automatisierung.png](https://static-docs.nocobase.com/a0d7c7c4fbe60d3f60b3a2a76155e99a.png)

**Ereignisgesteuerte Automatisierung**: Unterstützt ereignisbasierte Automatisierungsaufgaben, wie das automatische Auslösen von Workflows, wenn bestimmte Datenbedingungen erfüllt sind. Dieser ereignisgesteuerte Ansatz eignet sich für Szenarien, die Echtzeit-Reaktionen erfordern.

**Appsmith** bietet kein natives Workflow-System, aber Automatisierungsaufgaben können teilweise durch Skripte und Integrationen erreicht werden:

![Skriptbasierte Automatisierung.png](https://static-docs.nocobase.com/23509d18c8c31d72bd9cda3b6fb21e0d.png)

**Skriptbasierte Automatisierung**: Entwickler können JavaScript verwenden, um Datenoperationen und Seiteninteraktionen in Appsmith zu steuern. Dies ermöglicht zwar eine gewisse Automatisierung, erfordert jedoch Codierung, was komplexe Workflow-Konfigurationen schwieriger macht.

**Integration mit externen Automatisierungstools**: Appsmith unterstützt die Integration mit Tools wie Zapier oder Integromat, um die Automatisierungsfähigkeiten zu erweitern und Aufgaben wie automatische Datenverarbeitung, Benachrichtigungen und systemübergreifende Integrationen zu ermöglichen.

![Ereignisgesteuerte Seiteninteraktionen.png](https://static-docs.nocobase.com/50bcc73949d065b5bc22cedabc3d6477.png)

**Ereignisgesteuerte Seiteninteraktionen**: Appsmith unterstützt ereignisgesteuerte Aktionen, wie das Aktualisieren von Daten oder das Aufrufen von APIs nach dem Absenden eines Formulars. Diese Art der Automatisierung eignet sich für einfache Aufgaben auf Schnittstellenebene, bietet jedoch keine Unterstützung für komplexes Geschäftsprozessmanagement.

Im Vergleich dazu senkt der **visuelle Workflow-Designer von NocoBase** die Hürde für die Konfiguration komplexer Prozesse, während **Appsmith** für fortgeschrittenere Automatisierung Programmierkenntnisse erfordert.

## Bereitstellungsanalyse

**NocoBase Bereitstellungsoptionen**

![NocoBase Bereitstellungsoptionen.png](https://static-docs.nocobase.com/561a920abd18e1a751aa37c0e098bfcf.png)

NocoBase unterstützt drei verschiedene Installationsmethoden, jede mit leicht unterschiedlichen Bereitstellungsverfahren.

* [Docker](https://docs.nocobase.com/welcome/getting-started/deployment/docker-compose)
* [create-nocobase-app](https://docs.nocobase.com/welcome/getting-started/deployment/create-nocobase-app)
* [Git-Quellcode](https://docs.nocobase.com/welcome/getting-started/deployment/git-clone)

**Bereitstellungsschritte (Docker)**:

1. Installieren Sie Docker und Docker Compose.
2. Laden Sie die Docker-Compose-Datei von NocoBase herunter.
3. Führen Sie den Befehl `docker-compose up -d` aus.
4. Rufen Sie `http://localhost:13000` auf, um die Initialisierung abzuschließen.

**Systemanforderungen:**

* Empfohlen: 2-Kern-CPU, 4 GB RAM
* Unterstützt Linux, macOS und Windows

**Appsmith Bereitstellungsoptionen**

![Appsmith Bereitstellungsoptionen.png](https://static-docs.nocobase.com/bdb9a72ea6b41e21f2ba1ee5a8759b2f.png)

* **Docker-Bereitstellung**: Unterstützt Docker Compose oder Single-Container-Optionen, geeignet für die meisten Benutzer.
* **Kubernetes-Bereitstellung**: Ideal für groß angelegte Bereitstellungen.
* **Build aus dem Quellcode**: Ermöglicht hochgradig angepasste Setups.

**Bereitstellungsschritte (Docker-Methode):**

1. Installieren Sie Docker
2. Führen Sie den Appsmith-Container aus:

```Bash
docker run -d --name appsmith -p 80:80 -v "$PWD/stacks:/appsmith-stacks" appsmith/appsmith-ce
```

3. Rufen Sie http://localhost auf, um das Setup durchzuführen

**Systemanforderungen:**

* Minimum: 1-Kern-CPU, 4 GB RAM
* Empfohlen: 2-Kern-CPU, 8 GB RAM
* Unterstützt Linux, macOS und Windows

**Bereitstellungsvergleich**


| Funktion                    | NocoBase                | Appsmith                  |
| -------------------------- | ----------------------- | ------------------------- |
| Empfohlene Bereitstellung     | Docker                  | Docker                    |
| Bereitstellungsschwierigkeit      | Relativ einfach       | Mittel                  |
