## 14. NocoBase

![NocoBase1-8m9930.png](https://static-docs.nocobase.com/NocoBase1-8m9930.png)

### Pengenalan proyek

NocoBase adalah platform tanpa kode / rendah kode bertenaga AI sumber terbuka untuk membangun sistem bisnis internal perusahaan, panel admin, dan aplikasi bisnis yang membutuhkan iterasi berkelanjutan dengan cepat.

Seiring AI Agents secara bertahap memasuki skenario pengembangan dan bisnis, perusahaan membutuhkan lebih dari sekadar otomatisasi tugas tunggal. Mereka membutuhkan lingkungan sistem yang dapat mendukung data, izin, halaman, alur kerja, dan log aktivitas. Inilah fondasi yang disediakan NocoBase.

Di NocoBase, AI dapat berpartisipasi dalam pembangunan aplikasi. Pengguna dapat mendeskripsikan kebutuhan bisnis dalam bahasa alami, dan AI dapat membantu membuat model data, mengonfigurasi halaman, mengatur alur kerja, dan bahkan membantu pengembangan plugin. Setelah sistem dibuat, tim dapat terus memeriksa dan menyesuaikan bidang, hubungan, halaman, izin, dan alur kerja melalui antarmuka tanpa kode.

![NocoBase2-0l0hhf.png](https://static-docs.nocobase.com/NocoBase2-0l0hhf.png)

Karyawan AI dapat bekerja dengan data halaman, konteks bisnis, alat, dan basis pengetahuan untuk menangani analisis data, ringkasan konten, pembuatan laporan, terjemahan, ekstraksi konten tidak terstruktur, pengisian formulir, pemrosesan node alur kerja, dan banyak lagi. Dengan cara ini, AI tidak hanya membantu membangun sistem. Ia juga dapat terus berpartisipasi dalam operasi sistem.

![NocoBase3-xn1trm.png](https://static-docs.nocobase.com/NocoBase3-xn1trm.png)

NocoBase cocok untuk membangun sistem internal perusahaan yang membutuhkan model data, kontrol izin, orkestrasi alur kerja, dan pemeliharaan jangka panjang, terutama dalam skenario berikut:

* **Sistem bisnis internal perusahaan**: Manajemen pelanggan CRM, sistem persetujuan, sistem tiket, sistem manajemen proyek, backend operasi, backend admin, dan banyak lagi.

## 13. OpenCLI

![OpenCLI-qn3nn7.png](https://static-docs.nocobase.com/OpenCLI-qn3nn7.png)

Situs web resmi: [https://opencli.info](https://opencli.info/)

GitHub: [https://github.com/jackwener/OpenCLI](https://github.com/jackwener/OpenCLI)

Bintang GitHub: 24,1k

### Pengenalan proyek

OpenCLI mengonversi situs web, sesi browser, aplikasi Electron, dan alat lokal menjadi antarmuka CLI. Operasi yang awalnya membutuhkan antarmuka grafis dapat dibungkus sebagai perintah, memungkinkan manusia atau AI Agents untuk memanggil kemampuan web, menggunakan kembali sesi browser yang masuk, dan melakukan tindakan otomatis seperti navigasi, klik, pengisian formulir, dan ekstraksi data.

### Skenario yang cocok

OpenCLI cocok untuk skenario di mana tugas perlu diselesaikan melalui halaman web tetapi tidak ada API standar yang tersedia:

* **Otomatisasi web**: Membaca halaman backend, menanyakan informasi, mengisi formulir, dan mengekstrak data platform.
* **Antarmuka alat Agent**: Membungkus sesi browser dan kemampuan web sebagai perintah yang dapat dipanggil oleh AI Agents.

### Pengenalan proyek

Hermes Agent adalah AI Agent yang dapat meningkatkan diri sendiri yang dibangun oleh Nous Research. Ini secara resmi digambarkan sebagai "Agen yang tumbuh bersama Anda." Ini menekankan bahwa Agents dapat membuat dan meningkatkan keterampilan selama penggunaan berkelanjutan, merekam dan mencari percakapan masa lalu, dan secara bertahap memahami preferensi dan konteks jangka panjang pengguna.

### Skenario yang cocok

Hermes Agent cocok untuk skenario Agent yang membutuhkan operasi jangka panjang, memori jangka panjang, dan akumulasi Keterampilan:

* **Agent yang di-host sendiri**: Bangun Agent pribadi atau tim yang dapat berjalan terus menerus, merekam percakapan historis, dan menggunakan kembali keterampilan.
* **Asisten kolaborasi jangka panjang**: Melalui memori, keterampilan, dan pemanggilan alat, Agent secara bertahap dapat mengakumulasi konteks dan metode kerja di seluruh penggunaan berulang.

## Tanya Jawab Umum (FAQ)

### Q1: Bagaimana cara memilih di antara alat-alat ini?

A: Anda dapat terlebih dahulu mengidentifikasi jenis masalah apa yang ingin Anda selesaikan. Jika Anda ingin meningkatkan produktivitas pribadi, Anda dapat memprioritaskan Cherry Studio, Nanobot, Hermes Agent, dan SiYuan. Jika Anda mengembangkan AI Agents atau AI Copilot, Anda dapat melihat OpenClaude, CopilotKit, dan learn-claude-code. Jika Anda ingin Agents memanggil alat eksternal dan kemampuan web, Anda dapat melihat OpenCLI, Agent-Reach, dan Google Workspace CLI. Jika Anda membangun sistem bisnis perusahaan dan mengimplementasikan alur kerja, NocoBase layak untuk difokuskan.


| Tujuan                               | Alat yang direkomendasikan        |

### Q2: Bisakah tim non-teknis menggunakan alat AI Agent ini?

Ya, tetapi tergantung pada jenis alatnya.

Proyek seperti **OpenCLI, Agent-Reach, Google Workspace CLI, OpenClaude, CopilotKit, dan learn-claude-code** lebih merupakan alat CLI, kerangka kerja pengembangan, atau materi pembelajaran rekayasa Agent. Mereka membutuhkan beberapa dasar teknis.

Alat seperti **Cherry Studio dan SiYuan** memiliki antarmuka yang lebih ramah dan lebih mudah digunakan langsung oleh pengguna bisnis.

**NocoBase** berada di antara keduanya. Pengguna non-teknis dapat menggunakan sistem dan Karyawan AI yang telah dibangun. Pengguna dengan latar belakang teknis dapat lebih baik mengonfigurasi model data, halaman, izin, dan alur kerja. Dengan sumber daya pengembangan, tim juga dapat memperluasnya lebih dalam melalui plugin, API, Keterampilan, dan MCP.

### Q3: Apa yang paling harus diperhatikan perusahaan saat menggunakan AI Agents?

Ketika perusahaan menggunakan AI Agents, mereka tidak boleh hanya fokus pada kemampuan model. Mereka juga perlu mempertimbangkan data, izin, alur kerja, dan audit. Setelah Agent memasuki sistem bisnis, ia dapat mengakses data pelanggan, informasi pesanan, alur kerja persetujuan, konten kontrak, atau dokumen internal. Batasan yang jelas diperlukan.


| Masalah utama                         | Kemampuan yang harus difokuskan                                |
| ------------------------------------- | -------------------------------------------------------------- |
| Apakah data dapat dikendalikan?       | Hosting sendiri, izin database, cakupan akses data             |
| Apakah operasi dapat dilacak?         | Log audit, log aktivitas, catatan alur kerja                   |
| Apakah izin jelas?                    | Izin peran, izin bidang, kontrol cakupan data                  |
| Apakah tindakan kunci perlu konfirmasi? | Node konfirmasi manusia, alur kerja persetujuan, kondisi pemicu |
| Apakah output Agent stabil?           | Output terstruktur, mekanisme percobaan ulang, tinjauan manusia |

Nilai platform sistem bisnis seperti NocoBase adalah bahwa mereka menempatkan AI di dalam lingkungan sistem dengan izin, alur kerja, dan model data, daripada membiarkan Agents langsung menghadapi data yang tersebar dan operasi yang tidak terkendali. Untuk skenario seperti persetujuan, layanan pelanggan, entri data, dan pembuatan laporan, konfirmasi manusia dan catatan audit tetap sangat penting.

### Q4: Bisakah beberapa alat digabungkan?

Ya. Ekosistem AI Agent tidak dapat dicakup oleh satu alat. Banyak tim menggunakan alat yang berbeda pada lapisan yang berbeda: beberapa bertanggung jawab untuk membangun sistem, beberapa untuk memanggil halaman web, beberapa untuk menulis kode, beberapa untuk manajemen pengetahuan, dan beberapa untuk interaksi AI frontend.


| Skenario kombinasi                        | Kombinasi referensi                    |
| ----------------------------------------- | -------------------------------------- |
| Sistem bisnis + Karyawan AI               | NocoBase + Karyawan AI                 |
| Sistem bisnis + Coding Agent eksternal    | NocoBase + Claude Code / Codex / OpenCode |
| Operasi web + organisasi informasi        | OpenCLI + Agent-Reach                  |
| AI Copilot dalam produk                   | CopilotKit + API sistem bisnis         |
| Basis pengetahuan + Agent                 | SiYuan + Nanobot / Hermes Agent        |
| Meja kerja harian multi-model             | Cherry Studio + berbagai layanan model |

## Kesimpulan

Alat AI Agent dengan cepat bercabang ke arah yang berbeda. Beberapa alat berfokus pada pemanggilan model dan eksekusi tugas, beberapa berfokus pada otomatisasi web dan akses informasi eksternal, beberapa berfokus pada AI Coding dan Agent Harnesses, dan beberapa mulai memasuki sistem bisnis perusahaan dan manajemen proses jangka panjang.

Untuk pengguna individu, lebih baik memulai dengan skenario ringan seperti manajemen pengetahuan, asisten pribadi, AI Coding, atau otomatisasi browser. Untuk tim perusahaan, pertanyaan yang lebih penting adalah apakah Agent dapat memasuki alur kerja bisnis nyata dan berjalan secara stabil di bawah batasan data, izin, alur kerja, dan mekanisme audit.
