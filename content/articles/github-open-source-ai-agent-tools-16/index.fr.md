## 14. NocoBase

![NocoBase1-8m9930.png](https://static-docs.nocobase.com/NocoBase1-8m9930.png)

Site web officiel : [https://www.nocobase.com](https://www.nocobase.com/)

GitHub : [https://github.com/nocobase/nocobase](https://github.com/nocobase/nocobase)

Étoiles GitHub : 22,7k

### Présentation du projet

NocoBase est une plateforme no-code / low-code open source basée sur l'IA pour créer rapidement des systèmes métier internes d'entreprise, des panneaux d'administration et des applications métier nécessitant une itération continue.

Alors que les AI Agents entrent progressivement dans les scénarios de développement et d'affaires, les entreprises ont besoin de plus qu'une simple automatisation de tâches uniques. Elles ont besoin d'un environnement système capable de prendre en charge les données, les autorisations, les pages, les workflows et les journaux d'activité. C'est exactement le fondement que fournit NocoBase.

Dans NocoBase, l'IA peut participer à la création d'applications. Les utilisateurs peuvent décrire les besoins métier en langage naturel, et l'IA peut aider à créer des modèles de données, configurer des pages, orchestrer des workflows et même aider au développement de plugins. Une fois le système généré, les équipes peuvent continuer à vérifier et ajuster les champs, les relations, les pages, les autorisations et les workflows via l'interface no-code.

![NocoBase2-0l0hhf.png](https://static-docs.nocobase.com/NocoBase2-0l0hhf.png)

NocoBase prend également en charge la configuration d'Employés IA dans les systèmes métier. Les Employés IA peuvent travailler avec les données de page, le contexte métier, les outils et les bases de connaissances pour gérer l'analyse de données, les résumés de contenu, la génération de rapports, la traduction, l'extraction de contenu non structuré, le remplissage de formulaires, le traitement des nœuds de workflow, etc. De cette façon, l'IA n'aide pas seulement à construire le système. Elle peut également continuer à participer aux opérations du système.

![NocoBase3-xn1trm.png](https://static-docs.nocobase.com/NocoBase3-xn1trm.png)

### Scénarios adaptés

NocoBase est adapté à la création de systèmes internes d'entreprise nécessitant des modèles de données, un contrôle des autorisations, une orchestration de workflows et une maintenance à long terme, en particulier dans les scénarios suivants :

* **Systèmes métier internes d'entreprise** : Gestion de la relation client CRM, systèmes d'approbation, systèmes de tickets, systèmes de gestion de projet, backends opérationnels, backends d'administration, etc.

![NocoBase4-ngkai4.png](https://static-docs.nocobase.com/NocoBase4-ngkai4.png)

* **Modules ERP légers et de gestion métier** : Gestion des achats, gestion des stocks, gestion des commandes, gestion des équipements, gestion des actifs, gestion des archives, suivi client, etc.
* **Extension et amélioration des systèmes existants** : Pour les entreprises qui ont déjà des bases de données, ERP, CRM ou d'autres systèmes métier, NocoBase peut aider à intégrer les données, configurer les pages, gérer les autorisations et orchestrer les workflows.
* **Une base pour l'entrée des AI Agents dans les workflows métier** : NocoBase fournit aux AI Agents un environnement système métier clair, où les données métier ont des modèles, les opérations utilisateur ont des autorisations, les transitions de workflow ont des règles, et la maintenance ultérieure peut se poursuivre via la configuration no-code et les extensions de plugins.

## 13. OpenCLI

![OpenCLI-qn3nn7.png](https://static-docs.nocobase.com/OpenCLI-qn3nn7.png)

Site web officiel : [https://opencli.info](https://opencli.info/)

GitHub : [https://github.com/jackwener/OpenCLI](https://github.com/jackwener/OpenCLI)

Étoiles GitHub : 24,1k

### Présentation du projet

OpenCLI convertit les sites web, les sessions de navigateur, les applications Electron et les outils locaux en interfaces CLI. Les opérations qui nécessitaient à l'origine des interfaces graphiques peuvent être encapsulées en commandes, permettant aux humains ou aux AI Agents d'appeler des capacités web, de réutiliser des sessions de navigateur connectées et d'effectuer des actions automatisées telles que la navigation, les clics, le remplissage de formulaires et l'extraction de données.

### Scénarios adaptés

OpenCLI est adapté aux scénarios où les tâches doivent être accomplies via des pages web mais aucune API standard n'est disponible :

* **Automatisation web** : Lire les pages backend, interroger des informations, remplir des formulaires et extraire des données de plateforme.
* **Interface d'outil Agent** : Encapsuler les sessions de navigateur et les capacités web en commandes que les AI Agents peuvent appeler.

## 12. Agent-Reach

![Agent-Reach-ew3tec.png](https://static-docs.nocobase.com/Agent-Reach-ew3tec.png)

GitHub : [https://github.com/Panniantong/Agent-Reach](https://github.com/Panniantong/Agent-Reach)

Étoiles GitHub : 26,3k

### Présentation du projet

Agent-Reach se concentre sur l'accès aux informations externes pour les AI Agents. Via un outil CLI, il permet aux Agents de lire et de rechercher du contenu sur des plateformes telles que Twitter, Reddit, YouTube, GitHub, Bilibili et Xiaohongshu, résolvant le problème d'accès dans la récupération d'informations, la lecture de contenu et la recherche multiplateforme.

### Scénarios adaptés

Agent-Reach peut être utilisé pour le suivi des tendances technologiques, la recherche sur les concurrents, l'analyse des sentiments sur les réseaux sociaux, la recherche de sujets de contenu, l'organisation de projets GitHub et la collecte de matériel multiplateforme. Ce n'est pas en soi une plateforme Agent complète. Il ajoute plutôt des capacités d'accès aux informations externes aux Agents, donnant aux analyses, résumés et organisations ultérieurs une base d'informations plus riche.

### Scénarios adaptés

Hermes Agent est adapté aux scénarios Agent nécessitant un fonctionnement à long terme, une mémoire à long terme et une accumulation de compétences :

* **Agent auto-hébergé** : Construisez un Agent personnel ou d'équipe qui peut fonctionner en continu, enregistrer l'historique des conversations et réutiliser les compétences.
* **Assistant de collaboration à long terme** : Grâce à la mémoire, aux compétences et à l'appel d'outils, l'Agent peut progressivement accumuler le contexte et les méthodes de travail lors d'utilisations répétées.

### Q2 : Les équipes non techniques peuvent-elles utiliser ces outils AI Agent ?

Oui, mais cela dépend du type d'outil.

Les projets tels que **OpenCLI, Agent-Reach, Google Workspace CLI, OpenClaude, CopilotKit et learn-claude-code** sont davantage des outils CLI, des frameworks de développement ou des supports d'apprentissage d'ingénierie Agent. Ils nécessitent une certaine base technique.

Les outils tels que **Cherry Studio et SiYuan** ont des interfaces plus conviviales et sont plus faciles à prendre en main directement par les utilisateurs métier.

**NocoBase** se situe entre les deux. Les utilisateurs non techniques peuvent utiliser les systèmes et les Employés IA déjà construits. Les utilisateurs ayant une certaine base technique peuvent mieux configurer les modèles de données, les pages, les autorisations et les workflows. Avec des ressources de développement, les équipes peuvent également l'étendre plus profondément via des plugins, des API, des compétences et MCP.

### Q3 : À quoi les entreprises doivent-elles le plus prêter attention lors de l'utilisation des AI Agents ?

Lorsque les entreprises utilisent des AI Agents, elles ne doivent pas se concentrer uniquement sur les capacités du modèle. Elles doivent également prendre en compte les données, les autorisations, les workflows et l'audit. Une fois qu'un Agent entre dans un système métier, il peut accéder aux données client, aux informations de commande, aux workflows d'approbation, au contenu des contrats ou aux documents internes. Des limites claires sont nécessaires.


| Problème clé                          | Capacités sur lesquelles se concentrer                            |
| ------------------------------------- | ----------------------------------------------------------------- |
| Les données sont-elles contrôlables ? | Auto-hébergement, autorisations de base de données, périmètre d'accès aux données |
| Les opérations sont-elles traçables ? | Journaux d'audit, journaux d'activité, enregistrements de workflow |
| Les autorisations sont-elles claires ? | Autorisations de rôle, autorisations de champ, contrôle du périmètre des données |
| Les actions clés nécessitent-elles une confirmation ? | Nœuds de confirmation humaine, workflows d'approbation, conditions de déclenchement |
| La sortie de l'Agent est-elle stable ? | Sortie structurée, mécanismes de nouvelle tentative, révision humaine |

La valeur des plateformes de systèmes métier telles que NocoBase est qu'elles placent l'IA dans un environnement système avec des autorisations, des workflows et des modèles de données, plutôt que de laisser les Agents faire face directement à des données dispersées et à des opérations non contrôlées. Pour les scénarios tels que les approbations, le service client, la saisie de données et la génération de rapports, la confirmation humaine et les enregistrements d'audit restent très importants.

### Q4 : Plusieurs outils peuvent-ils être combinés ?

Oui. L'écosystème des AI Agents ne peut pas être couvert par un seul outil. De nombreuses équipes utilisent différents outils à différentes couches : certains sont responsables de la construction de systèmes, certains de l'appel de pages web, certains de l'écriture de code, certains de la gestion des connaissances, et certains de l'interaction IA frontend.


| Scénario de combinaison                     | Combinaison de référence                     |
| ------------------------------------------- | -------------------------------------------- |
| Système métier + Employé IA                 | NocoBase + Employé IA                        |
| Système métier + Agent de codage externe    | NocoBase + Claude Code / Codex / OpenCode    |
| Opération web + organisation d'informations | OpenCLI + Agent-Reach                        |
| AI Copilot dans le produit                  | CopilotKit + API système métier              |
| Base de connaissances + Agent               | SiYuan + Nanobot / Hermes Agent              |
| Poste de travail quotidien multi-modèles    | Cherry Studio + divers services de modèles   |

## Conclusion

Les outils AI Agent se ramifient rapidement dans différentes directions. Certains outils se concentrent sur l'appel de modèles et l'exécution de tâches, certains sur l'automatisation web et l'accès aux informations externes, certains sur le codage IA et les Harnais Agent, et certains commencent à entrer dans les systèmes métier d'entreprise et la gestion de processus à long terme.

Pour les utilisateurs individuels, il est préférable de commencer par des scénarios légers tels que la gestion des connaissances, les assistants personnels, le codage IA ou l'automatisation du navigateur. Pour les équipes d'entreprise, la question la plus importante est de savoir si un Agent peut entrer dans les workflows métier réels et fonctionner de manière stable sous les contraintes des données, des autorisations, des workflows et des mécanismes d'audit.
