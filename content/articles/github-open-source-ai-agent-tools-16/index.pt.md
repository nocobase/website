## 13. OpenCLI

![OpenCLI-qn3nn7.png](https://static-docs.nocobase.com/OpenCLI-qn3nn7.png)

Site oficial: [https://opencli.info](https://opencli.info/)

GitHub: [https://github.com/jackwener/OpenCLI](https://github.com/jackwener/OpenCLI)

Estrelas GitHub: 24,1k

### Introdução do projeto

OpenCLI converte sites, sessões de navegador, aplicativos Electron e ferramentas locais em interfaces CLI. Operações que originalmente exigiam interfaces gráficas podem ser encapsuladas como comandos, permitindo que humanos ou AI Agents chamem capacidades web, reutilizem sessões de navegador logadas e realizem ações automatizadas como navegação, cliques, preenchimento de formulários e extração de dados.

### Cenários adequados

OpenCLI é adequado para cenários onde as tarefas precisam ser concluídas através de páginas web, mas nenhuma API padrão está disponível:

* **Automação web**: Ler páginas de backend, consultar informações, preencher formulários e extrair dados da plataforma.
* **Interface de ferramenta do Agent**: Encapsular sessões de navegador e capacidades web como comandos que os AI Agents podem chamar.

## 12. Agent-Reach

![Agent-Reach-ew3tec.png](https://static-docs.nocobase.com/Agent-Reach-ew3tec.png)

GitHub: [https://github.com/Panniantong/Agent-Reach](https://github.com/Panniantong/Agent-Reach)

Estrelas GitHub: 26,3k

### Introdução do projeto

Agent-Reach foca no acesso a informações externas para AI Agents. Através de uma ferramenta CLI, permite que os Agents leiam e pesquisem conteúdo de plataformas como Twitter, Reddit, YouTube, GitHub, Bilibili e Xiaohongshu, resolvendo o problema de acesso na recuperação de informações, leitura de conteúdo e pesquisa entre plataformas.

### Cenários adequados

Agent-Reach pode ser usado para rastreamento de tendências de tecnologia, pesquisa de concorrentes, análise de sentimento em mídias sociais, pesquisa de tópicos de conteúdo, organização de projetos GitHub e coleta de materiais entre plataformas. Não é uma plataforma Agent completa por si só. Em vez disso, adiciona capacidades de acesso a informações externas aos Agents, dando às análises, resumos e organização posteriores uma base de informações mais rica.

## 1. Hermes Agent

## FAQ

### P1: Como devo escolher entre essas ferramentas?

R: Você pode primeiro identificar que tipo de problema deseja resolver. Se você quer melhorar a produtividade pessoal, pode priorizar Cherry Studio, Nanobot, Hermes Agent e SiYuan. Se você está desenvolvendo AI Agents ou AI Copilots, pode olhar para OpenClaude, CopilotKit e learn-claude-code. Se você quer que Agents chamem ferramentas externas e capacidades web, pode olhar para OpenCLI, Agent-Reach e Google Workspace CLI. Se você está construindo sistemas de negócios empresariais e implementando fluxos de trabalho, o NocoBase vale a pena focar.

### P2: Equipes não técnicas podem usar essas ferramentas de AI Agent?

Sim, mas depende do tipo de ferramenta.

Projetos como **OpenCLI, Agent-Reach, Google Workspace CLI, OpenClaude, CopilotKit e learn-claude-code** são mais como ferramentas CLI, frameworks de desenvolvimento ou materiais de aprendizado de engenharia de Agent. Eles exigem alguma base técnica.

Ferramentas como **Cherry Studio e SiYuan** têm interfaces mais amigáveis e são mais fáceis para usuários de negócios começarem a usar diretamente.

**NocoBase** fica entre os dois. Usuários não técnicos podem usar sistemas e Funcionários de IA que já foram construídos. Usuários com alguma formação técnica podem configurar melhor modelos de dados, páginas, permissões e fluxos de trabalho. Com recursos de desenvolvimento, as equipes também podem estendê-lo mais profundamente através de plugins, APIs, Habilidades e MCP.

### P3: No que as empresas devem prestar mais atenção ao usar AI Agents?

Quando as empresas usam AI Agents, elas não devem focar apenas nas capacidades do modelo. Elas também precisam considerar dados, permissões, fluxos de trabalho e auditoria. Uma vez que um Agent entra em um sistema de negócios, ele pode acessar dados de clientes, informações de pedidos, fluxos de trabalho de aprovação, conteúdo de contratos ou documentos internos. Limites claros são necessários.


| Questão chave                           | Capacidades a focar                                               |
| --------------------------------------- | ----------------------------------------------------------------- |
| Os dados são controláveis?              | Auto-hospedagem, permissões de banco de dados, escopo de acesso a dados |
| As operações são rastreáveis?           | Logs de auditoria, logs de atividade, registros de fluxo de trabalho |
| As permissões são claras?               | Permissões de função, permissões de campo, controle de escopo de dados |
| Ações-chave exigem confirmação?         | Nós de confirmação humana, fluxos de trabalho de aprovação, condições de gatilho |
| A saída do Agent é estável?             | Saída estruturada, mecanismos de repetição, revisão humana         |

O valor de plataformas de sistema de negócios como o NocoBase é que elas colocam a IA dentro de um ambiente de sistema com permissões, fluxos de trabalho e modelos de dados, em vez de deixar os Agents enfrentarem diretamente dados dispersos e operações não controladas. Para cenários como aprovações, atendimento ao cliente, entrada de dados e geração de relatórios, a confirmação humana e os registros de auditoria continuam sendo muito importantes.

### P4: Várias ferramentas podem ser combinadas?

Sim. O ecossistema de AI Agent não pode ser coberto por uma única ferramenta. Muitas equipes usam diferentes ferramentas em diferentes camadas: algumas são responsáveis por construir sistemas, algumas por chamar páginas web, algumas por escrever código, algumas por gestão de conhecimento e algumas por interação de IA no frontend.


| Cenário de combinação                      | Combinação de referência                  |
| ------------------------------------------ | ----------------------------------------- |
| Sistema de negócios + Funcionário de IA    | NocoBase + Funcionário de IA              |
| Sistema de negócios + Agent de Codificação externo | NocoBase + Claude Code / Codex / OpenCode |
| Operação web + organização de informações  | OpenCLI + Agent-Reach                     |
| AI Copilot no produto                      | CopilotKit + API do sistema de negócios   |
| Base de conhecimento + Agent               | SiYuan + Nanobot / Hermes Agent           |
| Bancada de trabalho diária multi-modelo    | Cherry Studio + vários serviços de modelo |

## Conclusão

As ferramentas de AI Agent estão rapidamente se ramificando em diferentes direções. Algumas ferramentas focam em chamada de modelo e execução de tarefas, algumas focam em automação web e acesso a informações externas, algumas focam em AI Coding e Agent Harnesses, e algumas estão começando a entrar em sistemas de negócios empresariais e gestão de processos de longo prazo.

Para usuários individuais, é melhor começar com cenários leves como gestão de conhecimento, assistentes pessoais, AI Coding ou automação de navegador. Para equipes empresariais, a questão mais importante é se um Agent pode entrar em fluxos de trabalho de negócios reais e ser executado de forma estável sob as restrições de dados, permissões, fluxos de trabalho e mecanismos de auditoria.
