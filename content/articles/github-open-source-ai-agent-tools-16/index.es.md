## 14. NocoBase

![NocoBase1-8m9930.png](https://static-docs.nocobase.com/NocoBase1-8m9930.png)

Sitio web oficial: [https://www.nocobase.com](https://www.nocobase.com/)

GitHub: [https://github.com/nocobase/nocobase](https://github.com/nocobase/nocobase)

Estrellas de GitHub: 22.7k

### Introducción del proyecto

NocoBase es una plataforma de código abierto sin código / low-code impulsada por IA para construir rápidamente sistemas empresariales internos, paneles de administración y aplicaciones comerciales que necesitan iteración continua.

A medida que los Agentes de IA ingresan gradualmente tanto en el desarrollo como en los escenarios comerciales, las empresas necesitan más que la automatización de tareas únicas. Necesitan un entorno de sistema que pueda soportar datos, permisos, páginas, flujos de trabajo y registros de actividad. Esta es exactamente la base que proporciona NocoBase.

En NocoBase, la IA puede participar en la construcción de aplicaciones. Los usuarios pueden describir los requisitos comerciales en lenguaje natural, y la IA puede ayudar a crear modelos de datos, configurar páginas, orquestar flujos de trabajo e incluso ayudar con el desarrollo de plugins. Después de que se genera el sistema, los equipos pueden continuar verificando y ajustando campos, relaciones, páginas, permisos y flujos de trabajo a través de la interfaz sin código.

![NocoBase2-0l0hhf.png](https://static-docs.nocobase.com/NocoBase2-0l0hhf.png)

NocoBase también admite la configuración de Empleados de IA dentro de los sistemas empresariales. Los Empleados de IA pueden trabajar con datos de página, contexto empresarial, herramientas y bases de conocimiento para manejar análisis de datos, resúmenes de contenido, generación de informes, traducción, extracción de contenido no estructurado, llenado de formularios, procesamiento de nodos de flujo de trabajo y más. De esta manera, la IA no solo ayuda a construir el sistema, sino que también puede continuar participando en las operaciones del sistema.

![NocoBase3-xn1trm.png](https://static-docs.nocobase.com/NocoBase3-xn1trm.png)

### Escenarios adecuados

NocoBase es adecuado para construir sistemas internos empresariales que requieren modelos de datos, control de permisos, orquestación de flujos de trabajo y mantenimiento a largo plazo, especialmente en los siguientes escenarios:

* **Sistemas empresariales internos**: Gestión de clientes CRM, sistemas de aprobación, sistemas de tickets, sistemas de gestión de proyectos, backends de operaciones, backends de administración y más.

![NocoBase4-ngkai4.png](https://static-docs.nocobase.com/NocoBase4-ngkai4.png)

* **Módulos de gestión empresarial y ERP ligero**: gestión de compras, gestión de inventario, gestión de pedidos, gestión de equipos, gestión de activos, gestión de archivos, seguimiento de clientes y más.
* **Extensión y mejora de sistemas existentes**: Para empresas que ya tienen bases de datos, ERP, CRM u otros sistemas comerciales, NocoBase puede ayudar a integrar datos, configurar páginas, gestionar permisos y orquestar flujos de trabajo.
* **Una base para que los Agentes de IA ingresen a los flujos de trabajo empresariales**: NocoBase proporciona a los Agentes de IA un entorno de sistema empresarial claro, donde los datos comerciales tienen modelos, las operaciones de los usuarios tienen permisos, las transiciones del flujo de trabajo tienen reglas y el mantenimiento posterior puede continuar a través de la configuración sin código y las extensiones de plugins.

## 13. OpenCLI

![OpenCLI-qn3nn7.png](https://static-docs.nocobase.com/OpenCLI-qn3nn7.png)

Sitio web oficial: [https://opencli.info](https://opencli.info/)

GitHub: [https://github.com/jackwener/OpenCLI](https://github.com/jackwener/OpenCLI)

Estrellas de GitHub: 24.1k

### Introducción del proyecto

OpenCLI convierte sitios web, sesiones de navegador, aplicaciones Electron y herramientas locales en interfaces CLI. Las operaciones que originalmente requerían interfaces gráficas se pueden envolver como comandos, lo que permite a humanos o Agentes de IA llamar capacidades web, reutilizar sesiones de navegador iniciadas y realizar acciones automatizadas como navegación, clics, llenado de formularios y extracción de datos.

### Escenarios adecuados

OpenCLI es adecuado para escenarios donde las tareas deben completarse a través de páginas web pero no hay una API estándar disponible:

* **Automatización web**: Leer páginas de backend, consultar información, llenar formularios y extraer datos de plataformas.
* **Interfaz de herramientas del Agente**: Envolver sesiones de navegador y capacidades web como comandos que los Agentes de IA pueden llamar.

### Escenarios adecuados

No es una plataforma de Agente completa por sí misma. En cambio, agrega capacidades de acceso a información externa a los Agentes, dando a los análisis, resúmenes y organización posteriores una base de información más rica.

### Introducción del proyecto

Hermes Agent es un Agente de IA que se automejora construido por Nous Research. Se describe oficialmente como "El agente que crece contigo". Enfatiza que los Agentes pueden crear y mejorar habilidades durante el uso continuo, registrar y buscar conversaciones pasadas, y comprender gradualmente las preferencias y el contexto a largo plazo de los usuarios.

### Escenarios adecuados

Hermes Agent es adecuado para escenarios de Agente que requieren operación a largo plazo, memoria a largo plazo y acumulación de Habilidades:

* **Agente autoalojado**: Construya un Agente personal o de equipo que pueda ejecutarse continuamente, registrar el historial de conversaciones y reutilizar habilidades.
* **Asistente de colaboración a largo plazo**: A través de la memoria, las habilidades y la llamada a herramientas, el Agente puede acumular gradualmente contexto y métodos de trabajo a lo largo del uso repetido.

## Preguntas Frecuentes

### P1: ¿Cómo debería elegir entre estas herramientas?

R: Primero puede identificar qué tipo de problema desea resolver. Si desea mejorar la productividad personal, puede priorizar Cherry Studio, Nanobot, Hermes Agent y SiYuan. Si está desarrollando Agentes de IA o Copilotos de IA, puede mirar OpenClaude, CopilotKit y learn-claude-code. Si desea que los Agentes llamen a herramientas externas y capacidades web, puede mirar OpenCLI, Agent-Reach y Google Workspace CLI. Si está construyendo sistemas empresariales e implementando flujos de trabajo, vale la pena centrarse en NocoBase.


| Objetivo                                 | Herramientas recomendadas         |
| ---------------------------------------- | --------------------------------- |
| Construir sistemas internos empresariales | NocoBase                          |
| Automatización web y llamada a herramientas | OpenCLI, Agent-Reach              |
| Automatización del sistema ofimático     | Google Workspace CLI              |
| Agente de terminal / Codificación de IA  | OpenClaude, learn-claude-code     |
| Copiloto de IA dentro del producto       | CopilotKit                        |
| Asistente de IA personal                 | Nanobot, Hermes Agent, CowAgent   |
| Banco de trabajo multi-modelo            | Cherry Studio                     |
| Gestión del conocimiento                 | SiYuan                            |
| Referencia de aplicación de Agente vertical | career-ops                        |

### P2: ¿Pueden los equipos no técnicos usar estas herramientas de Agente de IA?

Sí, pero depende del tipo de herramienta.

Proyectos como **OpenCLI, Agent-Reach, Google Workspace CLI, OpenClaude, CopilotKit y learn-claude-code** son más como herramientas CLI, marcos de desarrollo o materiales de aprendizaje de ingeniería de Agentes. Requieren cierta base técnica.

Herramientas como **Cherry Studio y SiYuan** tienen interfaces más amigables y son más fáciles de usar directamente para los usuarios comerciales.

**NocoBase** se encuentra entre ambos. Los usuarios no técnicos pueden usar sistemas y Empleados de IA que ya han sido construidos. Los usuarios con ciertos conocimientos técnicos pueden configurar mejor modelos de datos, páginas, permisos y flujos de trabajo. Con recursos de desarrollo, los equipos también pueden extenderlo más profundamente a través de plugins, APIs, Habilidades y MCP.

### P3: ¿A qué deberían prestar más atención las empresas al usar Agentes de IA?

Cuando las empresas usan Agentes de IA, no deben centrarse solo en las capacidades del modelo. También deben considerar los datos, los permisos, los flujos de trabajo y la auditoría. Una vez que un Agente ingresa a un sistema empresarial, puede acceder a datos de clientes, información de pedidos, flujos de trabajo de aprobación, contenido de contratos o documentos internos. Se requieren límites claros.


| Problema clave                          | Capacidades en las que centrarse                                  |
| --------------------------------------- | ----------------------------------------------------------------- |
| ¿Los datos son controlables?            | Autoalojamiento, permisos de base de datos, alcance de acceso a datos |
| ¿Las operaciones son rastreables?       | Registros de auditoría, registros de actividad, registros de flujo de trabajo |
| ¿Los permisos son claros?               | Permisos de rol, permisos de campo, control de alcance de datos   |
| ¿Las acciones clave requieren confirmación? | Nodos de confirmación humana, flujos de trabajo de aprobación, condiciones de activación |
| ¿La salida del Agente es estable?       | Salida estructurada, mecanismos de reintento, revisión humana     |

El valor de las plataformas de sistemas empresariales como NocoBase es que colocan la IA dentro de un entorno de sistema con permisos, flujos de trabajo y modelos de datos, en lugar de permitir que los Agentes se enfrenten directamente a datos dispersos y operaciones no controladas. Para escenarios como aprobaciones, servicio al cliente, entrada de datos y generación de informes, la confirmación humana y los registros de auditoría siguen siendo muy importantes.

### P4: ¿Se pueden combinar múltiples herramientas?

Sí. El ecosistema de Agentes de IA no puede ser cubierto por una sola herramienta. Muchos equipos usan diferentes herramientas en diferentes capas: algunas son responsables de construir sistemas, otras de llamar a páginas web, otras de escribir código, otras de la gestión del conocimiento y otras de la interacción de IA en el frontend.


| Escenario de combinación                     | Combinación de referencia                    |
| -------------------------------------------- | -------------------------------------------- |
| Sistema empresarial + Empleado de IA         | NocoBase + Empleado de IA                    |
| Sistema empresarial + Agente de Codificación externo | NocoBase + Claude Code / Codex / OpenCode    |
| Operación web + organización de información  | OpenCLI + Agent-Reach                        |
| Copiloto de IA dentro del producto           | CopilotKit + API del sistema empresarial     |
| Base de conocimiento + Agente                | SiYuan + Nanobot / Hermes Agent              |
| Banco de trabajo diario multi-modelo         | Cherry Studio + varios servicios de modelo   |

## Conclusión

Las herramientas de Agentes de IA se están ramificando rápidamente en diferentes direcciones. Algunas herramientas se centran en la llamada a modelos y la ejecución de tareas, algunas se centran en la automatización web y el acceso a información externa, algunas se centran en la Codificación de IA y los Arneses de Agente, y algunas están comenzando a ingresar a los sistemas empresariales y la gestión de procesos a largo plazo.

Para los usuarios individuales, es mejor comenzar con escenarios ligeros como la gestión del conocimiento, los asistentes personales, la Codificación de IA o la automatización del navegador. Para los equipos empresariales, la pregunta más importante es si un Agente puede ingresar a los flujos de trabajo comerciales reales y ejecutarse de manera estable bajo las restricciones de datos, permisos, flujos de trabajo y mecanismos de auditoría.
