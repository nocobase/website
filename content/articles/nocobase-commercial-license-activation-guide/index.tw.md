## **NocoBase 商業授權啟用指南**

為了提升用戶體驗並加強安全性，我們全面升級了商業授權與啟用流程。新的授權方式取代了過去在設定檔中輸入使用者名稱和密碼的做法，引入了基於「實例 ID」和「授權金鑰」的機制。

新方案的核心優勢包括：

*   **增強安全性**：不再需要以明文儲存您的帳號資訊，降低敏感資訊外洩的風險。
*   **簡化流程**：授權直接綁定您的 NocoBase 實例，讓授權管理與更新更加直觀便捷。
*   **提高靈活性**：當您的部署環境變更時，可以輕鬆重新產生授權，快速適應新環境。
*   **授權保護**：透過後台授權管理與狀態監控，您可以監控授權的實際使用情況，及時發現並處理未經授權的應用，防止商業授權被濫用。

本指南將引導您完成全新的 NocoBase 商業授權啟用流程。

## **授權啟用逐步指南**

開始前，請確保您已成功安裝並正在運行 NocoBase 社群版。

### **步驟 1：進入授權頁面並複製實例 ID**

1.  在您的本地 NocoBase 應用程式中，導航至 `系統設定` -> `授權設定` 頁面。
2.  在此頁面上，您會看到一個 **實例 ID** 字串，這是您當前運行環境的唯一識別碼。點擊其右側的複製圖示。

![img_v3_02nv_a5e3edcd-d411-4dc9-8b1c-d3e29631243g](https://static-docs.nocobase.com/img_v3_02nv_a5e3edcd-d411-4dc9-8b1c-d3e29631243g.jpg)

### **步驟 2：在服務平台上建立應用程式並關聯實例 ID**

1.  登入 NocoBase 服務平台，在「應用程式」管理頁面上點擊「新增」按鈕。
2.  在彈出的表單中，填寫您的應用程式資訊，並將上一步複製的 `實例 ID` 貼到 `實例 ID` 輸入欄位中，然後提交。

![20250707233019](https://static-docs.nocobase.com/20250707233019.png)

### **步驟 3：取得授權金鑰**

1.  應用程式建立成功後，返回授權列表，找到對應的條目，並點擊操作欄中的「檢視授權」按鈕。
2.  在授權詳細資訊彈出視窗中，點擊「複製」按鈕以取得與您實例匹配的 **授權金鑰**。
   ![img_v3_02nv_138161f1-8ac4-4458-81c0-8a4d0f702d8g](https://static-docs.nocobase.com/img_v3_02nv_138161f1-8ac4-4458-81c0-8a4d0f702d8g.jpg)

### **步驟 4：在 NocoBase 中設定授權金鑰**

1.  返回您的本地 NocoBase `授權設定` 頁面。
2.  將剛剛複製的 `授權金鑰` 貼到 `授權金鑰` 輸入欄位中，然後點擊「提交」按鈕。
3.  儲存成功後，系統會顯示「授權金鑰已成功儲存，請重新啟動伺服器」，表示授權金鑰已設定成功，等待重啟。

![img_v3_02nv_45050542-c4dd-45b7-afdf-4a4e9fb75f4g](https://static-docs.nocobase.com/img_v3_02nv_45050542-c4dd-45b7-afdf-4a4e9fb75f4g.jpg)

### **步驟 5：完成啟用並下載商業版及商業插件**

重新啟動您的 NocoBase 服務以套用授權，並自動下載商業版及商業插件。

*   **對於 Docker 部署**：執行重啟指令 `docker compose restart app`。
*   **對於原始碼或 `create-nocobase-app` 部署**：在專案根目錄下執行 `yarn nocobase pkg download-pro` 指令。

服務重啟並成功下載插件後，您的商業授權即已啟用。如果您需要更換授權，可以重新進入授權設定頁面，點擊「更換金鑰」按鈕，並重複上述流程。

## **如何檢視授權資訊**

授權啟用後，您可以隨時點擊 NocoBase 介面右上角的授權資訊圖示，檢視當前授權的詳細狀態，包括授權版本、到期日期、網域匹配狀態以及已授權的插件列表。

## 部署與授權快速檢查清單

遇到問題時，您可以參考此檢查清單進行快速排查：

- [ ]  **步驟 1：取得實例 ID**
  - [ ]  NocoBase 社群版/未授權版已成功運行。
  - [ ]  已從 `授權設定` 頁面完整複製 `實例 ID`。
- [ ]  **步驟 2：產生授權金鑰**
  - [ ]  已在服務平台上建立應用程式。
  - [ ]  建立應用程式時貼上的 `實例 ID` 是最新且正確的。
  - [ ]  已從「檢視授權」中完整複製 `授權金鑰`。
- [ ]  **步驟 3：啟用**
  - [ ]  `授權金鑰` 已貼回 NocoBase 並成功儲存。
  - [ ]  **已根據部署方式執行容器重啟或 `download-pro` 指令。** （許多用戶會漏掉此步驟！）
- [ ]  **驗證**
  - [ ]  授權資訊彈出視窗顯示狀態為「啟用中」。

## 常見問題 (FAQ)

在安裝或使用過程中，您可能會遇到以下提示。請依照對應的解決方案處理：

### 1. `License key mismatch` - 授權金鑰與環境不匹配

<image src="https://static-docs.nocobase.com/undefined20250701193351673.png" />

[問]為什麼設定後會出現 `License key mismatch`？

[答]您的運行環境可能已變更，導致「`實例 ID`」更新，但您的應用程式仍持有舊的「`授權金鑰`」。
**解決方案**：請嚴格按照本指南的**步驟 1**，從您的**當前環境**取得最新的 `實例 ID`，然後完成後續流程以產生新的「金鑰」。

### 2. `Invalid license key` - 授權金鑰已過期

<image src="https://static-docs.nocobase.com/undefined20250701193558012.png" />

[問]`Invalid license key` 是什麼意思？

[答]您的 `授權金鑰` 可能已過期，或因其他原因被平台標記為無效。
**解決方案**：請聯繫我們或在服務平台上檢查您的授權狀態。如有需要，請重新產生 `授權金鑰` 並進行設定。

### 3. `Invalid license key format` - 授權金鑰格式錯誤

<image src="https://static-docs.nocobase.com/undefined20250701193148501.png" />

[問]遇到 `Invalid license key format` 該怎麼辦？

[答]您貼上的 `授權金鑰` 不完整或格式不正確，導致系統無法解析。
**解決方案**：請返回服務平台，確保您已完整複製 `授權金鑰`，然後重新貼上並儲存。

### 4. 存取系統時強制彈出授權資訊對話框

<image src="https://static-docs.nocobase.com/undefined20250625164238000.png" width="400" />

[問]為什麼授權資訊對話框總是出現？

[答]此彈窗通常表示授權驗證失敗。可能的原因包括：

*   `授權金鑰` 已損壞或與本地環境不匹配
*   授權已過期或無效
*   `授權金鑰` 與您運行的商業插件版本不匹配

**解決方案**：最直接的解決方案是**完全重新執行授權設定流程**：取得最新的 `實例 ID`，產生新的 `授權金鑰`，並在本地系統中設定。

### 5. 環境遷移相關問題

[問]我遷移了伺服器/資料庫，商業插件停止工作了。該怎麼辦？

[答]這與上述問題相同。伺服器或資料庫是構成「身份證 (`實例 ID`)」的關鍵資訊，當它們變更時，`實例 ID` 也會變更。

**解決方案**：同上，取得新的 `實例 ID`，產生新的 `授權金鑰`，然後重新啟用。整個過程就像搬家後更新身份證地址一樣。

### 6. 插件未生效

[問]我已經重啟了，為什麼商業插件仍然無法使用？

[答]如果您已確認 `授權金鑰` 設定正確，請嘗試以下排查：

1.  **檢查網路**：確保您的伺服器可以存取 NocoBase 的 npm 倉庫。
2.  **檢查日誌**：查看 NocoBase 啟動日誌中是否有插件下載失敗或授權驗證錯誤的資訊。
3.  **手動執行**：如果您使用的是原始碼部署，請再次手動執行 `yarn nocobase pkg download-pro` 指令，並觀察指令行輸出。

### 7. 在運行中的系統上啟用後，商業插件未顯示？

**[問]** 我先啟動了 NocoBase 系統，然後在設定中輸入了授權金鑰。啟用成功了，為什麼商業插件不出現？

**[答]** 這是預期行為，也是一個非常常見的問題。NocoBase 的設計是在**服務啟動時**檢查授權並載入相應的插件。如果您在系統已運行時完成啟用，則需要**重新啟動 NocoBase 服務**來觸發商業插件的下載和啟用。

簡單來說，授權就像 NocoBase 的「通行證」，但它需要「重新進門」（重啟）才能檢查並使用這張通行證。

請根據您的部署方式執行重啟操作：

*   **對於 Docker 部署**：執行重啟指令 `docker compose restart app`。
*   **對於原始碼或 `create-nocobase-app` 部署**：首先，在您的終端機中停止正在運行的服務（通常按 `Ctrl+C`），然後再次運行啟動指令，例如 `yarn start`。

### 8. 新舊授權方式比較

[問]這個流程與之前的使用者名稱/密碼方式有何不同？

[答]舊方法會將您的帳號密碼直接暴露在環境中，安全性不足。新的 `授權金鑰` 系統有以下幾個優點：

1.  **更安全**：您無需暴露主帳號密碼，僅使用綁定特定環境的金鑰。
2.  **更靈活**：您可以為不同的客戶或專案產生不同的 `授權金鑰`，管理更方便。
3.  **更清晰**：授權資訊一目了然，所有詳細記錄都在服務平台上。
