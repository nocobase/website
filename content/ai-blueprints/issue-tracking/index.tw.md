## 介紹

使用您常用的 AI Agent 與 NocoBase，快速打造一套可客製、可靠且能長期持續迭代的問題追蹤系統，用於管理專案、Epic、問題、子任務、狀態、優先層級、負責人和問題編號。

您可以直接複製下方提示詞，讓 AI Agent 在 NocoBase 中生成問題追蹤系統的基礎架構，再透過無程式碼介面調整欄位、頁面、任務層級和處理流程。

此系統適合軟體開發、產品研發、基礎設施、API、前端和技術支援團隊，用於統一管理 Bug、技術任務、改善需求和跨專案問題。

## 問題追蹤系統能解決哪些問題？

問題追蹤讓團隊清楚掌握問題屬於哪個專案、目前進展如何、優先程度多高、由誰負責，以及它與其他任務之間有什麼層級關係。

當問題散落在聊天記錄、文件和多個專案工具中時，團隊很容易出現重複提交、負責人不明、緊急問題長期停滯，以及 Epic 與具體執行任務脫節等情形。

系統支援 Todo、In Progress、In Review、Done、Backlog 和 Cancelled 等狀態，以及 Urgent、High、Medium、Low 和 None 等優先層級。團隊可以透過左側篩選器或頂部狀態標籤，快速定位需要處理的問題。

每筆問題記錄還能顯示負責人和唯一的問題編號，方便團隊在討論、提交程式碼或回顧時精確引用。

AI 團隊負責人可以分析每個問題的內容，識別所需技能，並推薦合適的員工負責。AI 資料分析員工還能讀取全部問題資料，生成完成率、活躍問題、緊急問題風險和負責人工作量等分析報告。

## 核心功能

### 專案與問題管理

- **多專案統一管理：** 集中管理 Core Platform、API Gateway、Frontend 和 Infrastructure 等專案的問題。
- **問題基本資訊：** 記錄問題標題、編號、所屬專案、狀態、優先層級、負責人和所屬 Epic。
- **專案快速篩選：** 依專案查看問題清單和問題數量，快速定位不同團隊的待處理事項。

### Epic 與子任務層級

- **層級化任務管理：** 使用 Epic 組織大型需求，並在其下關聯 Bug、任務和子問題。
- **展開與收合檢視：** 依需求展開或收合子任務，在整體目標和具體執行項目之間快速切換。
- **執行進度追蹤：** 結合父子問題的狀態，查看 Epic 下各項工作的推進情形。

### 狀態與優先層級管理

- **完整問題狀態：** 支援 Backlog、Todo、In Progress、In Review、Done 和 Cancelled 等階段。
- **多層次優先層級：** 使用 Urgent、High、Medium、Low 和 None 區分問題的處理緊迫程度。
- **風險快速識別：** 透過狀態和優先層級標籤，定位緊急但尚未完成或長期停滯的問題。

### 問題工作台與篩選

- **多條件組合篩選：** 支援依專案、狀態、優先層級和負責人篩選問題。
- **問題快捷操作：** 可新增、刪除、重新整理問題，並展開或收合全部任務層級。
- **統一問題清單：** 在同一工作台查看問題標題、狀態、優先層級、負責人和問題編號。

### AI 智慧指派

- **AI 分析任務需求：** 根據問題標題和內容判斷完成任務所需的技能。
- **AI 推薦負責人：** 結合員工能力和目前工作量，推薦合適的執行人員。
- **AI 輔助任務協調：** 發現無人負責或分配不合理的問題，並提供調整建議。

### AI 問題與風險分析

- **問題進度分析：** 統計問題總數、完成率和目前活躍問題數量。
- **緊急問題識別：** 分析 Urgent 問題的完成情形，定位仍處於處理中的高風險任務。
- **工作量與指派分析：** 比較不同負責人承擔的問題數量，並識別未指派的問題。
- **風險摘要生成：** 自動彙整專案目前的主要風險、阻塞點和優先處理事項。

## 為何選擇用 AI 與 NocoBase 打造問題追蹤系統？

問題追蹤系統真正投入研發團隊使用後，需要處理的不只是標題和狀態，還包括專案、Epic、子任務、優先層級、負責人和資料分析。

如果用一般的 Vibe Coding 從零生成一個任務清單，通常可以快速完成新增、刪除、修改、查詢，但層級任務、複合篩選、負責人指派、資料權限和分析報告仍然需要持續開發。

NocoBase 能將專案、問題、Epic、子任務和人員資料關聯在一起，並透過樹狀清單、篩選器、狀態標籤和優先層級標籤統一呈現。團隊也可以根據實際研發流程新增專案、狀態或優先層級規則。

AI 可以進一步參與任務指派和資料分析。AI 團隊負責人可以理解問題內容，並根據員工技能推薦合適的執行人；AI 資料分析員工可以讀取問題資料，自動彙整完成率、風險、優先層級分佈和團隊工作量。

如此打造出來的問題追蹤系統，不只是一個靜態任務表，而是一個能支援複雜研發協作、持續分析和長期迭代的內部系統。

## FAQ

1. **可以依專案、狀態和優先層級快速篩選問題嗎？**

可以。系統支援依 Core Platform、API Gateway、Frontend、Infrastructure 等專案篩選問題，也可以組合篩選 Backlog、Todo、In Progress、In Review、Done、Cancelled 等狀態，以及 Urgent、High、Medium、Low、None 等優先層級。

每個篩選項還可以顯示對應的問題數量，協助團隊快速判斷問題主要集中在哪個專案或處理階段。

2. **可以管理 Epic 和子任務之間的層級關係嗎？**

可以。複雜需求可以先建立為 Epic，再將 Bug、技術任務或改善事項作為子任務關聯到對應 Epic 下。

工作台支援展開和收合層級，讓團隊既能查看整體目標，也能追蹤每個具體執行項目的狀態、優先層級和負責人。

3. **可以根據團隊流程自訂問題狀態嗎？**

可以。系統可以使用 Backlog、Todo、In Progress、In Review、Done 和 Cancelled 等狀態，也可以根據團隊實際流程新增測試中、待發布、已阻塞等階段。

當研發流程發生變化時，可以持續調整狀態、頁面篩選和工作流程，不需要重新搭建整個系統。

4. **可以為不同問題設定優先層級並識別高風險任務嗎？**

可以。每筆問題都可以設定為 Urgent、High、Medium、Low 或 None，並透過不同標籤清楚顯示。

團隊可以結合狀態和優先層級識別高風險任務，例如仍停留在 In Progress 的緊急問題，或長期未進入處理階段的高優先層級問題。

5. **AI 可以根據問題內容推薦合適的負責人嗎？**

可以。AI 團隊負責人可以分析問題標題、描述和所屬專案，判斷任務所需的技術能力，再結合員工技能和目前工作量推薦合適的執行人。

這樣既能減少人工指派時間，也能協助團隊發現任務是否集中在少數成員身上。

6. **可以查看每位成員承擔了多少問題嗎？**

可以。系統可以依負責人統計問題數量，並進一步區分已完成、處理中和待處理任務。

AI 還能識別工作量是否分配不均、是否存在無人負責的問題，以及哪些成員承擔了較多緊急或高優先層級任務。

7. **AI 可以分析問題的完成率和研發風險嗎？**

可以。AI 可以讀取問題狀態、優先層級、專案和負責人資料，統計問題總數、完成率和活躍問題比例，並識別需要重點關注的任務。

例如，截圖中的分析報告可以發現 Urgent 問題完成率偏低、仍有多筆緊急任務處於 In Progress，以及目前是否存在未指派問題。

8. **問題分析報告可以預覽和匯出嗎？**

可以。AI 生成報告後，可以透過 Preview、Markdown 和 HTML 查看。

報告也可以下載為 Markdown、HTML，或列印為 PDF，適合用於研發週會、專案回顧和管理層簡報。

9. **可以控制不同成員能查看和修改哪些問題嗎？**

可以。NocoBase 支援依角色、專案、負責人和操作類型設定權限。

例如，一般成員只能更新指派給自己的問題；專案負責人可以管理所屬專案的問題和負責人；團隊負責人可以查看跨專案工作量；管理員可以設定欄位、狀態、優先層級和權限。

這比所有成員都能修改全部問題更適合真實的研發協作環境。

10. **可以追蹤問題被誰修改過嗎？**

可以依需求啟用操作歷程和稽核日誌，記錄問題狀態、優先層級、負責人和其他欄位的變更。

當緊急任務被延期、取消或重新指派時，團隊可以追蹤誰在何時進行了修改，方便回顧和責任追溯。

11. **Claude Code、Codex、Cursor、OpenCode 可以協助搭建問題追蹤系統嗎？**

可以。Claude Code、Codex、Cursor、OpenCode 等 AI coding agent 可以接入 NocoBase，根據提示詞生成專案資料表、問題資料表、Epic 與子任務關聯、篩選頁面和分析功能。

基礎系統生成後，團隊仍可透過 NocoBase 的無程式碼介面持續調整資料結構、頁面、狀態和權限，不需要每次變動都讓 AI 從零重新撰寫應用程式。

12. **這套系統和一般 Vibe Coding 生成的任務清單有什麼不同？**

一般 Vibe Coding 可以快速生成一個問題清單或看板，但面對 Epic 層級、複合篩選、精細權限、操作記錄、工作量分析和長期維護時，通常還需要不斷補充程式碼。

NocoBase 已提供業務系統所需的資料關聯、權限、工作流程、稽核和擴充能力。AI 用於加快搭建和分析，NocoBase 則負責承載持續運作的研發流程。

13. **這套問題追蹤系統適合研發團隊長期使用嗎？**

適合。系統上線後，可以持續新增專案、問題類型、狀態、優先層級、分析指標和自動化流程，並根據團隊規模調整權限。

企業還可以依需求啟用單一登入（SSO）、通知、API、操作歷程和外掛程式擴充。相較於一次性的任務管理展示頁，更適合建立可維護、可追蹤且能持續迭代的問題追蹤系統。
